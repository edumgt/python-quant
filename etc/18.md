### 1. 의사결정나무(Decision Tree) 원리

**쉽게 이해하기**
- **의사결정나무(Decision Tree) 원리** 개념은 `입력값 확인 → 규칙 적용 → 결과 해석` 순서로 단계별로 보면 쉽습니다.
- 먼저 작은 데이터로 검증한 뒤 실제 데이터에 적용하세요.

**미니 예시**
```python
# 의사결정나무(Decision Tree) 원리 미니 예시
values = [100, 103, 101, 106]
delta = values[-1] - values[0]
print('시작 대비 변화:', delta)
```

### 2. 과적합(Overfitting)과 가지치기(Pruning)

**쉽게 이해하기**
- **과적합(Overfitting)과 가지치기(Pruning)** 개념은 `입력값 확인 → 규칙 적용 → 결과 해석` 순서로 단계별로 보면 쉽습니다.
- 먼저 작은 데이터로 검증한 뒤 실제 데이터에 적용하세요.

**미니 예시**
```python
# 과적합(Overfitting)과 가지치기(Pruning) 미니 예시
values = [100, 103, 101, 106]
delta = values[-1] - values[0]
print('시작 대비 변화:', delta)
```

### 3. 랜덤 포레스트(Random Forest) 원리

**쉽게 이해하기**
- **랜덤 포레스트(Random Forest) 원리** 개념은 `특징 만들기 → 규칙 학습 → 라벨 예측` 3단계로 보면 쉽습니다.
- 모델 성능은 정확도 하나만 보지 말고 오분류 사례를 같이 점검하세요.

**미니 예시**
```python
# 랜덤 포레스트(Random Forest) 원리 미니 예시
scores = [0.2, 0.7, 0.9, 0.4]
labels = ['매도' if s < 0.5 else '매수' for s in scores]
print('분류 결과:', labels)
```

### 4. 특징 중요도(Feature Importance) 분석

**쉽게 이해하기**
- **특징 중요도(Feature Importance) 분석** 개념은 `입력값 확인 → 규칙 적용 → 결과 해석` 순서로 단계별로 보면 쉽습니다.
- 먼저 작은 데이터로 검증한 뒤 실제 데이터에 적용하세요.

**미니 예시**
```python
# 특징 중요도(Feature Importance) 분석 미니 예시
values = [100, 103, 101, 106]
delta = values[-1] - values[0]
print('시작 대비 변화:', delta)
```

### 5. 실습: Random Forest로 주가 예측 및 팩터 분석

**쉽게 이해하기**
- **실습: Random Forest로 주가 예측 및 팩터 분석** 개념은 `입력(x)와 출력(y)의 관계 찾기`라고 생각하면 쉽습니다.
- 예측값만 보지 말고, 실제값과의 차이(오차)를 같이 확인하세요.

**미니 예시**
```python
# 실습: Random Forest로 주가 예측 및 팩터 분석 미니 예시
x = [1, 2, 3, 4]
y = [2, 4, 6, 8]
slope = (y[-1] - y[0]) / (x[-1] - x[0])
next_y = y[-1] + slope
print('다음 값 예측:', next_y)
```

# Chapter 04. Random Forest 분류 결과 읽는 법

> 💡 **쉽게 이해하기**: 전문가 한 명의 의견보다 전문가 100명의 다수결이 더 믿을 만합니다. 랜덤 포레스트는 수백 개의 결정 트리가 각자 예측하고 다수결로 최종 답을 내는 앙상블 모델입니다. 단일 트리보다 훨씬 안정적이고 과적합에 강합니다.

---

`RandomForest.py`는 트리 앙상블 기반의 대표 분류 모델을 다룹니다.

### 모델 이해
- 여러 의사결정트리의 다수결로 최종 클래스 결정
- 단일 트리보다 과적합에 강하고 성능이 안정적인 편

### 출력 지표
- Accuracy: 전체 정답 비율
- Precision/Recall/F1: 클래스별 성능 균형 확인

### 실무 확장 팁
- 가상 데이터 대신 CSV를 연결
- 범주형 인코딩, 결측치 처리 파이프라인 추가
- API 응답에 feature importance를 포함해 설명 가능성 강화
