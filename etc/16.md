### 1. 머신러닝 분류 (지도, 비지도, 강화학습)

**쉽게 이해하기**
- **머신러닝 분류 (지도, 비지도, 강화학습)** 개념은 `특징 만들기 → 규칙 학습 → 라벨 예측` 3단계로 보면 쉽습니다.
- 모델 성능은 정확도 하나만 보지 말고 오분류 사례를 같이 점검하세요.

**미니 예시**
```python
# 머신러닝 분류 (지도, 비지도, 강화학습) 미니 예시
scores = [0.2, 0.7, 0.9, 0.4]
labels = ['매도' if s < 0.5 else '매수' for s in scores]
print('분류 결과:', labels)
```

### 2. 데이터 분할: 훈련/검증/테스트 세트

**쉽게 이해하기**
- **데이터 분할: 훈련/검증/테스트 세트** 개념은 `입력값 확인 → 규칙 적용 → 결과 해석` 순서로 단계별로 보면 쉽습니다.
- 먼저 작은 데이터로 검증한 뒤 실제 데이터에 적용하세요.

**미니 예시**
```python

values = [100, 103, 101, 106]
delta = values[-1] - values[0]
print('시작 대비 변화:', delta)
```

### 3. 선형 회귀(Linear Regression) 이론

**쉽게 이해하기**
- **선형 회귀(Linear Regression) 이론** 개념은 `입력(x)와 출력(y)의 관계 찾기`라고 생각하면 쉽습니다.
- 예측값만 보지 말고, 실제값과의 차이(오차)를 같이 확인하세요.

**미니 예시**
```python
# 선형 회귀(Linear Regression) 이론 미니 예시
x = [1, 2, 3, 4]
y = [2, 4, 6, 8]
slope = (y[-1] - y[0]) / (x[-1] - x[0])
next_y = y[-1] + slope
print('다음 값 예측:', next_y)
```

### 4. 비용함수(Loss Function)와 경사하강법

**쉽게 이해하기**
- **비용함수(Loss Function)와 경사하강법** 개념은 작은 변수와 함수부터 직접 실행해보면 가장 빨리 익숙해집니다.
- 한 번에 길게 작성하기보다, 한 줄씩 출력 결과를 확인하면서 진행하세요.

**미니 예시**
```python
# 비용함수(Loss Function)와 경사하강법 미니 예시
name = 'quant'
scores = [80, 90, 100]
avg = sum(scores) / len(scores)
print(f'{name} 평균 점수: {avg:.1f}')
```

### 5. 실습: 주가 예측 선형 회귀 모델 (scikit-learn)

**쉽게 이해하기**
- **실습: 주가 예측 선형 회귀 모델 (scikit-learn)** 개념은 `입력(x)와 출력(y)의 관계 찾기`라고 생각하면 쉽습니다.
- 예측값만 보지 말고, 실제값과의 차이(오차)를 같이 확인하세요.

**미니 예시**

# Chapter 02. Cross Validation을 쉽게 이해하기

> 💡 **쉽게 이해하기**: 시험을 한 번만 보면 운이 작용할 수 있습니다. K-Fold는 데이터를 5등분하여 5번 번갈아 시험을 보고 평균 점수를 내는 방법입니다. 한 번의 우연보다 훨씬 공정하게 모델 성능을 평가할 수 있습니다.

---

`CrossValid.py`의 핵심은 "한 번 나눠서 평가하지 말고 여러 번 나눠 평균을 보자"입니다.

### 왜 필요한가?
- 단일 train/test split은 운에 따라 점수가 흔들릴 수 있음
- K-Fold는 데이터를 K개로 나눠 번갈아 검증하여 안정적인 평균 점수 제공

### 코드에서 보는 포인트
- `make_classification`: 실습용 분류 데이터 자동 생성
- `LogisticRegression`: 빠르고 해석이 쉬운 선형 분류기
- `cross_val_score(..., cv=5)`: 5번 반복 검증

### API로 확장 시 장점
- 실시간으로 `n_samples`, `n_features`, `cv`를 바꿔 결과 비교 가능
- 프론트에서 하이퍼파라미터 실험 UI를 만들기 쉬움
