## 학습 내용

### 1. strategy() 함수로 백테스트 전환

- `indicator()` 대신 `strategy()`를 사용하면 동일한 시그널 로직을 실제 매매 시뮬레이션으로 검증할 수 있습니다.
- 기본 선언 예시:

```pinescript
strategy("Moving Average Crossover Strategy", overlay=true, initial_capital=100000)
```

- 핵심 전환 체크:
  - 진입/청산 조건을 `strategy.entry`, `strategy.exit`, `strategy.close`로 분리
  - 시각화용 plot과 체결 로직을 분리하여 관리
  - 재도색(repaint) 가능성을 줄이기 위해 확정봉 기준 조건 사용

### 2. 거래비용 설정 및 포지션 사이징

- 백테스트 신뢰도를 높이기 위해 수수료, 슬리피지, 주문 수량 규칙을 반드시 명시합니다.
- 설정 포인트:
  - `commission_type`, `commission_value`로 수수료 반영
  - `slippage`로 체결 오차 반영
  - 고정 수량(`qty`) 또는 비율 기반(`qty_percent`) 포지션 사이징 선택
- 동일 전략도 비용 가정에 따라 성과가 크게 바뀌므로, 낙관/중립/보수 시나리오를 비교합니다.

### 3. TradingView 백테스트 결과 분석

- Strategy Tester에서 아래 지표를 우선 점검합니다.
  - 순이익(Net Profit), 최대낙폭(Max Drawdown), 승률(Win Rate), 손익비(Avg Win / Avg Loss)
  - 거래 횟수(Trades)와 포지션 보유 시간
- 해석 원칙:
  - 순이익만 보지 않고 손실 구간의 길이/깊이 확인
  - 특정 구간 과최적화 여부를 표본 외(`out-of-sample`) 기간으로 점검
  - 거래 수가 과도하게 적으면 통계적 신뢰도 부족 가능성 검토

### 4. 실습: 커스텀 인디케이터 전략 TradingView 백테스트

- 실습 절차:
  1. 기존 인디케이터 신호(예: 이동평균 교차)를 진입/청산 조건으로 정의
  2. 거래비용과 포지션 사이징을 포함해 `strategy()` 스크립트 완성
  3. 최근 2~3년 구간과 변동성 급등 구간을 나눠 결과 비교
  4. 성과 지표를 표로 정리하고 개선 아이디어 2개 이상 도출
- 제출 기준:
  - 전략 코드
  - 파라미터 설정 근거
  - 성과 요약(순이익, MDD, 승률, 거래 수)
