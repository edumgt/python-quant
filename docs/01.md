# 개인·법인·세무·회계 기초 상식

>
> 📝 **한자 병기 및 어원 사전**: 이 문서에 등장하는 용어의 한자·어원·일제강점기 유래는 → [voca.md](voca.md)

퀀트·투자 분석을 하다 보면 기업의 **재무제표·세금·법적 구조**를 이해하는 것이 필수입니다. 기업이 왜 이런 회계처리를 하는지, 세금을 얼마나 내는지를 모르면 재무 분석이 왜곡됩니다.

## 들어가기 — 운용의 기본 개념

**운용(運用)**은 돈을 그냥 보관하는 것이 아니라, **투자를 통해 돈을 불리는 활동**입니다.

- 예금은 원금 보존 중심, 운용은 수익 창출 중심입니다.
- 운용의 핵심은 수익률을 높이면서도 손실 위험을 관리하는 균형입니다.
- 즉, 좋은 운용은 “많이 버는 것”과 “크게 잃지 않는 것”을 함께 추구합니다.

### Tenbagger(텐배거)란?

**Tenbagger**는 투자 원금이 **10배(10-bagger)**가 된 종목 또는 투자 성과를 뜻합니다.

- 100만 원이 1,000만 원이 되면 텐배거입니다.
- 장기 복리, 기업 성장, 분산 투자, 리스크 관리가 텐배거 가능성을 높입니다.
- 단기 급등만 노리기보다, 재무·산업·경쟁력을 함께 보는 분석이 중요합니다.

## A. 사람과 법인 — 법적 주체의 구분

> 📺 [🎬 사람과 법인 법적 주체의](https://www.youtube.com/results?search_query=사람과+법인+법적+주체의+구분+한국어)

### ① 자연인(個人) vs 법인(法人)

| 구분 | 자연인 (Natural Person) | 법인 (Legal Person / Corporation) |
|------|------------------------|-----------------------------------|
| 의미 | 태어나면서 권리·의무의 주체가 되는 사람 | 법이 사람처럼 권리·의무 능력을 부여한 **인위적 존재** |
| 사례 | 홍길동 씨 (개인) | 삼성전자 주식회사, 사단법인 XX협회 |
| 설립 방법 | 출생 | 설립 등기 (법원·등기소) |
| 사망/소멸 | 자연사 | 해산·청산 |
| 세금 | 소득세 (종합소득세·근로소득세) | 법인세 |
| 재산 | 본인 재산 = 개인 재산 | 법인 재산 ≠ 주주 재산 (분리) |
| 채무 책임 | 본인 전 재산으로 무한 책임 | 원칙적으로 법인 재산 한도 내 |

> 💡 법인이 **별도의 법적 인격**을 가진다는 것이 핵심입니다. 삼성전자가 빚을 져도, 이재용 회장이 개인 재산으로 갚을 의무는 없습니다(주주 유한 책임).

---

### ② 개인사업자 vs 법인사업자

사업을 할 때는 **개인사업자**와 **법인사업자(회사)** 중 하나를 선택합니다.

| 구분 | 개인사업자 | 법인사업자 |
|------|-----------|-----------|
| 법적 주체 | 자연인(사장 본인) | 별도 법인 |
| 설립 절차 | 사업자 등록 (세무서, 간단) | 법인 설립 등기 (법원) + 사업자 등록 |
| 최저 자본금 | 없음 | 없음 (단, 주식회사는 실무상 최소 100만원 이상) |
| 소득세 vs 법인세 | **종합소득세** (6~45% 누진) | **법인세** (9~24%, 낮은 세율) |
| 대표자 급여 | 비용 처리 **불가** (사장 소득 = 사업 소득) | 비용 처리 **가능** (급여로 처리) |
| 채무 책임 | **무한 책임** — 사업 빚 = 개인 빚 | **유한 책임** — 출자액 한도 |
| 대출·투자 유치 | 상대적으로 어려움 | 주식 발행·외부 투자 유치 용이 |
| 폐업 절차 | 간단 (폐업 신고) | 복잡 (청산·등기 말소) |
| 적합 상황 | 소규모 자영업·초기 1인 사업 | 성장을 목표로 한 사업, 투자 유치 필요 시 |

> 💡 **절세 전략**: 연 소득이 약 5,000만원을 초과하면 소득세 세율이 법인세보다 높아지기 시작합니다. 이 시점에 법인 전환을 검토하는 경우가 많습니다.

### ③ 기업 규모 분류(대기업·중소기업·소상공인) 기준

실무에서는 기업 규모를 **단일 숫자 하나**로 나누지 않고, 법령 목적에 따라 기준을 달리 적용합니다.

| 구분 | 일반적 판단 기준(요약) | 실무 포인트 |
|------|------------------------|------------|
| **대기업** | 공정거래법상 **상호출자제한기업집단/공시대상기업집단** 등으로 분류되는 대규모 기업집단 중심 | 계열사 범위, 내부거래·공시 규제, 지배구조 공시 의무가 중요 |
| **중소기업** | 중소기업기본법상 업종별 매출액 기준 + 자산·독립성 요건 충족 | 정책자금·세제감면·보증지원 대상 여부를 먼저 확인 |
| **소상공인** | 소상공인기본법상 **상시근로자 수(업종별 5인/10인 미만)** 및 매출 요건 충족 | 지역신보·소상공인시장진흥공단 지원사업 적용 여부가 핵심 |

> ⚠️ 기준은 법령 개정과 업종에 따라 달라질 수 있으므로, 실제 업무에서는 중소벤처기업부 고시·공정위 공시의 최신 기준을 반드시 확인해야 합니다.

## B. 회사의 종류 — 책임 구조에 따른 분류

> 📺 [🎬 회사의 종류 책임 구조에](https://www.youtube.com/results?search_query=회사의+종류+책임+구조에+따른+분류+한국어)

상법은 회사를 **책임 방식**에 따라 5종으로 구분합니다.

| 회사 형태 | 사원의 책임 | 특징 | 실제 사용 |
|-----------|-----------|------|----------|
| **합명회사** | **전원 무한 책임** | 소수 신뢰 관계, 인적 회사 | 거의 없음 |
| **합자회사** | 무한책임사원 + 유한책임사원 혼합 | 투자자 보호 일부 가능 | 드묾 |
| **유한책임회사** | **전원 유한 책임** | 미국 LLC와 유사, 내부 자유도 높음 | 스타트업·VC |
| **유한회사** | **전원 유한 책임** | 사원 50인 이하, 비공개 | 중소기업 |
| **주식회사** | **전원 유한 책임** | 주식 발행, 공개·상장 가능 | **가장 보편적** |

### ① 무한책임 (Unlimited Liability)

사업체의 채무를 **개인 재산으로도 끝까지 갚아야** 하는 책임 구조입니다.

```
사업 빚 1억원 발생
  └── 무한책임 → 개인 통장·집·자동차까지 전부 상환 의무
```

- 해당: 개인사업자, 합명회사 사원, 합자회사 무한책임사원
- 위험: 사업 실패 시 개인 파산으로 이어질 수 있음

### ② 유한책임 (Limited Liability)

**출자한 금액 한도 내에서만** 책임지는 구조입니다.

```
주식 1,000만원어치 투자
  └── 유한책임 → 최대 손실 = 1,000만원 (개인 재산 추가 손실 없음)
```

- 해당: 주식회사 주주, 유한회사·유한책임회사 사원
- 의의: 위험 부담을 투자액으로 제한 → 대규모 자본 조달 가능

> 💡 유한책임 덕분에 수백만 명이 삼성전자 주식을 살 수 있습니다. 회사가 망해도 주가가 0원이 될 뿐, 개인이 회사 빚을 갚지 않아도 됩니다.

## C. 주식회사 (株式會社, Corporation)

> 📺 [🎬 주식회사](https://www.youtube.com/results?search_query=주식회사+한국어)

### ① 주식회사의 구조

```
주주 (출자자·소유자)
  │  출자 (주식 매입)
  ▼
주식회사 (법인)
  │  선임
  ▼
이사회 (Board of Directors)
  │  경영 위임
  ▼
대표이사 (CEO) + 경영진
```

| 기관 | 역할 |
|------|------|
| **주주총회** | 최고 의사결정 기관. 이사 선임, 합병, 정관 변경 등 |
| **이사회** | 경영 감독. 주요 업무 결의 |
| **대표이사** | 회사 대표, 일상 경영 집행 |
| **감사** | 이사 업무·회계 감시 |

### ② 주식 (株式, Share)

| 항목 | 내용 |
|------|------|
| 의미 | 주식회사 소유권(지분)을 균등하게 나눈 단위 |
| 종류 | 보통주 (의결권 O) / 우선주 (배당 우선, 의결권 X) |
| 액면가 | 정관에 기재된 주당 명목 금액 (보통 100원·500원·5,000원) |
| 시가 | 시장에서 거래되는 실제 가격 |
| 배당 | 이익 일부를 주주에게 분배 |
| 의결권 | 1주 = 1표 (보통주 기준) |

### ③ 상장 vs 비상장

| 구분 | 상장(公開)회사 | 비상장(私的)회사 |
|------|--------------|----------------|
| 주식 거래 | 거래소(KRX)에서 자유롭게 매매 | 장외·개인간 거래 |
| 공시 의무 | DART 공시 의무 (강함) | 거의 없음 |
| 자금 조달 | IPO·유상증자로 불특정 다수에게 | 제한적 |
| 대표 예시 | 삼성전자, 카카오 | 카카오 (상장 전), 비상장 중소기업 |

### ④ 주식회사의 공시 방법·공시 내용 수준·예시

주식회사의 공시는 투자자가 회사 상태를 비교적 공정하게 알 수 있도록 만드는 **공식 정보 공개 절차**입니다.

| 구분 | 무엇을 통해 공개하나 | 투자자가 보는 포인트 |
|------|----------------------|----------------------|
| **정기공시** | DART(전자공시시스템): 사업보고서·반기보고서·분기보고서 | 매출·이익 추세, 부채비율, 현금흐름, 감사의견 |
| **수시공시** | DART 및 거래소(KIND) 공시 | 합병·분할·대규모 투자·소송·경영권 변경 등 즉시성 이슈 |
| **IR/보도자료** | 회사 홈페이지 IR, 실적발표 자료 | 법적 공시 보완용 참고자료(핵심 수치는 공시와 대조 필요) |

#### 공시 내용의 수준(신뢰도·구속력) 이해

| 수준 | 특징 | 예시 |
|------|------|------|
| **높음 (법정 공시)** | 법령 기반 제출, 허위·누락 시 제재 가능, 투자 판단의 기준 자료 | 사업보고서, 분기보고서, 주요사항보고서 |
| **중간 (거래소 공시)** | 상장 규정 기반 공시, 시장에 빠르게 전달 | 조회공시 답변, 공정공시, 불성실공시 관련 공지 |
| **보완 (IR·인터뷰)** | 이해를 돕는 설명 자료이나 법정 공시 대비 구속력은 낮음 | 실적발표 슬라이드, 경영진 인터뷰 |

> 💡 실무 팁: **IR 자료에서 본 숫자·표현은 반드시 DART 원문 공시로 교차 확인**하는 습관이 중요합니다.

#### 초보 투자자용 공시 읽기 예시

1. **분기보고서 확인**: 최근 4개 분기 매출·영업이익이 꾸준한지 먼저 본다.  
2. **수시공시 확인**: 같은 기간에 유상증자·전환사채·대규모 차입 공시가 있었는지 본다.  
3. **해석 결합**: 실적은 개선인데 자금조달 공시가 잦다면, 성장 투자인지 유동성 악화 신호인지 추가 점검한다.

### ⑤ 지배구조(企業支配構造, Corporate Governance)의 의미

**지배구조**는 회사에서 누가 의사결정을 하고, 누가 이를 감시·견제하는지에 대한 구조입니다.

| 구성 요소 | 핵심 질문 | 투자자 관점 |
|----------|----------|------------|
| **소유 구조** | 최대주주·특수관계인 지분이 어떻게 구성되는가? | 지배력 집중이 과도하면 소수주주 권익 침해 위험 증가 |
| **이사회 구조** | 사외이사·전문위원회가 실질적으로 작동하는가? | 경영진 견제 기능이 약하면 의사결정 왜곡 가능 |
| **감사·내부통제** | 회계·준법 통제가 독립적으로 운영되는가? | 분식회계·내부자 거래 리스크를 조기 점검 가능 |
| **주주권 보호** | 배당정책·공시·의결권 절차가 투명한가? | 장기적으로 기업가치 할인(코리아 디스카운트) 완화에 중요 |

> 💡 같은 실적의 기업이라도 지배구조가 투명한 회사는 자본조달 비용이 낮고, 장기 밸류에이션에서 프리미엄을 받는 경향이 있습니다.

### ⑥ 법인의 상태 변화 — 파산·회생·회생절차·상장폐지

회사는 항상 잘 되는 것만은 아닙니다. 몸이 아프면 병원에 가듯, 회사도 상태가 나빠지면 단계가 바뀝니다.

| 상태 | 쉬운 뜻 | 한자/영단어 | 초등생 예시 |
|------|---------|-------------|------------|
| **파산** | 빚을 갚을 힘이 거의 없어, 문을 닫는 마지막 상태 | 破産 (Bankruptcy) | 문구점이 돈을 못 갚아 가게를 완전히 닫는 상황 |
| **회생** | 완전히 끝내지 않고, 살릴 방법을 찾아 다시 일어나는 것 | 回生 (Rehabilitation / Turnaround) | 아픈 화분에 물·햇빛을 다시 맞춰서 살려보는 것 |
| **회생절차** | 법원이 정한 규칙에 따라 빚을 조정하고 회사를 살리는 공식 과정 | 回生節次 (Rehabilitation Procedure) | 담임선생님이 규칙표를 만들어 반 친구들 갈등을 순서대로 해결하는 것 |
| **상장폐지** | 주식시장에서 그 회사 주식을 더 이상 사고팔 수 없게 되는 것 | 上場廢止 (Delisting) | 학교 장터에서 팔던 쿠폰이 규칙 위반으로 장터 목록에서 빠지는 것 |

#### 한눈에 흐름 보기

```
경영 악화
  ├─ 회생 시도 (스스로 구조조정)
  ├─ 회생절차 (법원 관리 아래 정상화 시도)
  └─ 실패하면 파산 가능

상장회사라면 별도로
  └─ 거래소 기준 미달 시 상장폐지 가능
```

#### 투자자 관점에서 아주 쉽게

- **파산**: 남은 돈을 나눠 갚아도 부족할 수 있어 주주 손실이 매우 큽니다.
- **회생**: 살아날 가능성은 있지만 시간이 오래 걸리고 불확실성이 큽니다.
- **회생절차**: 법원 관리로 질서는 생기지만, 기존 주식 가치가 크게 줄 수 있습니다.
- **상장폐지**: 시장에서 바로 매매가 어려워져 현금화가 매우 힘들어질 수 있습니다.

> 💡 쉬운 기억법: **파산=끝**, **회생=살리기 도전**, **회생절차=법원 규칙으로 살리기**, **상장폐지=시장 거래 정지에 가까운 퇴장**.

## D. 세금 — 개인과 법인의 차이

> 📺 [🎬 세금 개인과 법인의 차이](https://www.youtube.com/results?search_query=세금+개인과+법인의+차이+한국어)

### ① 소득세 (所得稅, Income Tax) — 개인

| 과세표준 | 세율 | 누진공제 |
|---------|------|---------|
| 1,400만원 이하 | **6%** | — |
| 1,400~5,000만원 | **15%** | 126만원 |
| 5,000~8,800만원 | **24%** | 576만원 |
| 8,800만원~1.5억원 | **35%** | 1,544만원 |
| 1.5억~3억원 | **38%** | 1,994만원 |
| 3억~5억원 | **40%** | 2,594만원 |
| 5억~10억원 | **42%** | 3,594만원 |
| 10억원 초과 | **45%** | 6,594만원 |

> 지방소득세 10% 추가 → 최고 세율 실질 **49.5%**

```python
def income_tax(taxable_income: int) -> int:
    """종합소득세 계산 (2024년 기준, 지방소득세 제외)"""
    brackets = [
        (14_000_000,   0.06,          0),
        (50_000_000,   0.15,  1_260_000),
        (88_000_000,   0.24,  5_760_000),
        (150_000_000,  0.35, 15_440_000),
        (300_000_000,  0.38, 19_940_000),
        (500_000_000,  0.40, 25_940_000),
        (1_000_000_000,0.42, 35_940_000),
        (float('inf'), 0.45, 65_940_000),
    ]
    for limit, rate, deduction in brackets:
        if taxable_income <= limit:
            return int(taxable_income * rate - deduction)
    return 0

for income in [30_000_000, 80_000_000, 200_000_000, 500_000_000]:
    tax = income_tax(income)
    print(f"소득 {income:>12,}원 → 세금 {tax:>11,}원 (실효세율 {tax/income*100:.1f}%)")
```

### ② 법인세 (法人稅, Corporate Tax) — 법인

| 과세표준 | 세율 |
|---------|------|
| 2억원 이하 | **9%** |
| 2억~200억원 | **19%** |
| 200억~3,000억원 | **21%** |
| 3,000억원 초과 | **24%** |

> 지방소득세 10% 추가 → 최고 실질 세율 **26.4%**

```python
def corporate_tax(taxable_income: int) -> int:
    """법인세 계산 (2024년 기준)"""
    brackets = [
        (200_000_000,       0.09,           0),
        (20_000_000_000,    0.19,  20_000_000),
        (300_000_000_000,   0.21,  60_000_000),
        (float('inf'),      0.24, 150_000_000),
    ]
    for limit, rate, deduction in brackets:
        if taxable_income <= limit:
            return int(taxable_income * rate - deduction)
    return 0
```

> 💡 **법인 전환 시점**: 과세표준이 약 5,000만원을 넘으면 소득세(24%)가 법인세(9~19%)보다 높아지기 시작합니다.

## E-A. 조세의 분류

> 📺 [🎬 조세의 분류](https://www.youtube.com/results?search_query=조세의+분류+한국어)

조세(租稅)는 **국가 또는 지방자치단체가 재정 수입을 위해 법률에 따라 강제적으로 징수하는 금전급부**입니다. 분류 기준에 따라 다양하게 나뉩니다.

---

### ① 과세 주체(부과 주체)에 따른 분류

세금을 **누가 부과·징수하느냐**에 따라 나눕니다.

| 구분 | 정의 | 주요 세목 |
|------|------|-----------|
| **국세 (國稅)** | 중앙정부(국가)가 부과·징수 | 소득세, 법인세, 부가가치세, 상속세, 증여세, 개별소비세, 종합부동산세 |
| **지방세 (地方稅)** | 지방자치단체(시·도, 시·군·구)가 부과·징수 | 취득세, 재산세, 지방소득세, 지방교육세, 주민세 |

```
세금
├── 국세 (국가 → 세무서·국세청)
│   ├── 내국세: 소득세, 법인세, 부가가치세, 상속·증여세 등
│   └── 관세: 수입품에 부과
└── 지방세 (지자체 → 시청·구청)
    ├── 도세: 취득세, 지방소비세
    └── 시·군세: 재산세, 주민세, 지방소득세
```

> 💡 지방소득세는 소득세·법인세 산출세액의 **10%**를 별도로 납부합니다. 예: 소득세 1,000만 원 → 지방소득세 100만 원 추가.

---

### ② 전가(轉嫁) 여부에 따른 분류 — 직접세 vs 간접세

세금 부담이 **납세자 본인에게 귀착되느냐, 타인에게 전가되느냐**에 따라 나눕니다.

| 구분 | 정의 | 특징 | 주요 세목 |
|------|------|------|-----------|
| **직접세 (Direct Tax)** | 납세자(세금 내는 사람) = 담세자(실제 부담자) | 소득·재산에 비례, 누진 적용 가능 | 소득세, 법인세, 상속세, 증여세, 종합부동산세 |
| **간접세 (Indirect Tax)** | 납세자 ≠ 담세자 (판매자가 내지만 소비자가 부담) | 물가에 포함되어 최종 소비자가 실질 부담 | 부가가치세(VAT), 개별소비세, 주세, 담배소비세 |

```
[직접세 예시] 소득세
  월급쟁이 → 소득세 원천징수 → 국가
  납세자 = 담세자 (본인 부담)

[간접세 예시] 부가가치세(VAT)
  소비자 → 상품 구매 시 VAT 포함 가격 지불
    → 판매자(납세자)가 국가에 납부
  납세자(판매자) ≠ 담세자(소비자)
```

> 💡 간접세는 소득 수준에 관계없이 동일한 금액을 부담하므로 **역진성**(저소득자에게 상대적으로 부담이 큰 현상)이 나타납니다.

> ✅ **금융소득(이자·배당)은 직접세**로 분류되며, 기본적으로 **이자소득세/배당소득세 14% + 지방소득세 1.4%(총 15.4%)**가 원천징수됩니다.  
> 연간 금융소득(이자+배당) 합계가 **2,000만 원 초과**하면 종합소득에 합산해 종합과세합니다.

### ③ 과세 대상(과세 물건)에 따른 분류

**무엇에 세금을 매기느냐**에 따라 나눕니다.

| 과세 유형 | 정의 | 세목 예시 |
|-----------|------|-----------|
| **소득세 계열** | 소득(벌어들인 돈)에 부과 | 소득세, 법인세, 지방소득세 |
| **재산세 계열** | 보유 재산(자산)에 부과 | 재산세, 종합부동산세, 상속세, 증여세 |
| **소비세 계열** | 소비(지출·구매)에 부과 | 부가가치세, 개별소비세, 주세, 담배소비세 |
| **유통세 계열** | 재산·권리의 이전(거래)에 부과 | 취득세, 인지세, 증권거래세 |

```
투자자 관련 세금 흐름 예시
  주식 매입 → 증권거래세 (유통세)
  주식 보유 → 배당소득세 (소득세)
  주식 매도 차익 → 금융투자소득세/양도소득세 (소득세)
  부동산 취득 → 취득세 (유통세)
  부동산 보유 → 재산세·종부세 (재산세)
  부동산 양도 → 양도소득세 (소득세)
```

### ④ 세율 구조에 따른 분류

**세율이 어떻게 적용되느냐**에 따라 나눕니다.

| 구분 | 정의 | 특징 | 예시 |
|------|------|------|------|
| **누진세 (Progressive Tax)** | 과세표준이 클수록 높은 세율 적용 | 소득 재분배 효과 | 소득세 (6~45%), 법인세 (9~24%) |
| **비례세 (Proportional Tax)** | 과세표준에 관계없이 동일 세율 | 단순·명확 | 부가가치세 (10%), 취득세 (4%) |
| **역진세 (Regressive Tax)** | 과세표준이 커질수록 실효세율 하락 | 저소득층 상대적 부담 증가 | 일부 간접세 (소득 대비) |

```python
# 누진세 계산 예시 — 2024년 기준 소득세
def income_tax(income):
    brackets = [
        (14_000_000, 0.06),
        (50_000_000, 0.15),
        (88_000_000, 0.24),
        (150_000_000, 0.35),
        (300_000_000, 0.38),
        (500_000_000, 0.40),
        (1_000_000_000, 0.42),
        (float('inf'), 0.45),
    ]
    tax, prev = 0, 0
    for limit, rate in brackets:
        if income <= prev:
            break
        taxable = min(income, limit) - prev
        tax += taxable * rate
        prev = limit
    return tax

print(f"5천만원 소득세: {income_tax(50_000_000):,.0f}원")   # 약 624만원
print(f"1억원 소득세:   {income_tax(100_000_000):,.0f}원")  # 약 2,010만원
```

### ⑥ 납세 방법에 따른 분류

**세금을 어떻게 결정하고 납부하느냐**에 따라 나눕니다.

| 구분 | 정의 | 예시 |
|------|------|------|
| **신고납세 (Self-Assessment)** | 납세자가 스스로 계산·신고·납부 | 소득세(종합소득세 신고), 법인세, 부가가치세 |
| **부과납세 (Government Assessment)** | 과세관청이 세액을 결정·고지하면 납부 | 재산세, 종합부동산세, 자동차세 |
| **원천징수 (Withholding Tax)** | 지급자가 지급 시 세금을 미리 떼고 납부 | 근로소득세, 이자·배당소득세(15.4%), 프리랜서(3.3%) |

```
[원천징수 흐름]
  회사(지급자) → 직원 급여에서 소득세 미리 공제
    → 세무서에 납부 (다음달 10일까지)
    → 직원은 연말정산으로 정산
```

## E-B. 일상생활 속 세금 상식

> 📺 [🎬 일상생활 속 세금 상식](https://www.youtube.com/results?search_query=일상생활+속+세금+상식+한국어)

> 세금은 투자·사업에만 관련된 것이 아닙니다. 월급을 받고, 커피를 사고, 차를 사고, 집을 사는 **모든 일상** 속에 세금이 녹아 있습니다.

---

### ① 월급쟁이의 세금 — 근로소득세와 연말정산

직장인의 세금은 **회사가 매월 급여에서 미리 원천징수**하고, 연말에 실제 세액과 비교해 **환급 or 추가납부**합니다.

```
[월급 흐름]
  총급여
  − 비과세(식대 월 20만원, 차량유지비 월 20만원 등)
  = 과세 급여
  − 각종 공제 (인적공제, 신용카드, 의료비, 교육비, 기부금 등)
  = 과세표준
  × 세율(6~45% 누진)
  = 산출세액
  − 세액공제
  = 결정세액
  ↔ 기납부세액(원천징수 합계) 비교
  → 환급 or 추가납부
```

**핵심 공제 항목 요약**

| 공제 항목 | 한도·조건 | 절세 팁 |
|-----------|-----------|---------|
| **신용카드·체크카드** | 총급여 25% 초과분의 15~40% 공제 | 체크카드·현금영수증이 공제율 높음 |
| **의료비** | 총급여 3% 초과분의 15% (본인·장애인은 한도 없음) | 본인 의료비는 한도 없이 공제 |
| **교육비** | 본인 전액, 자녀 1인당 300만원 | 학원비는 제외(학교·교육기관만) |
| **보험료** | 보장성보험 연 100만원 한도 15% 공제 | 자동차보험, 실손보험 포함 |
| **연금저축·IRP** | 연금저축 600만원 + IRP 합산 900만원 한도, 세액공제 13~16.5% | 소득이 낮을수록 공제율 높음 |
| **기부금** | 법정·지정 기부금 15~30% 공제 | 고향사랑기부제: 10만원 이하 전액 세액공제 |

> 💡 **13월의 월급**: 공제 항목을 잘 챙기면 수십만~수백만원 환급 가능. 반대로 소득공제 누락 시 추가납부가 발생합니다.

### ② 프리랜서·부업 소득의 세금

| 구분 | 원천징수 | 신고 의무 | 비고 |
|------|----------|-----------|------|
| **프리랜서 소득** | 3.3% 원천징수 (소득세 3% + 지방소득세 0.3%) | 5월 종합소득세 신고 | 필요경비 인정 가능 |
| **유튜브·블로그 수익** | 별도 원천징수 없음 | 5월 종합소득세 신고 | 연 100만원 초과 시 신고 의무 |
| **직장인 부업 소득** | 프리랜서와 동일 | 근로소득 + 사업소득 합산 신고 | 연 300만원 초과 시 합산 |
| **중고 거래** | 없음 | 반복·영리 목적 시 신고 대상 | 생활용품 단순 처분은 비과세 |

```
[프리랜서 절세 포인트]
  수입 100만원 × 3.3% = 3.3만원 원천징수
  → 5월 신고 시 필요경비(장비, 교통비, 통신비 등) 공제
  → 실제 납부 세금 줄어들 수 있음 (환급 가능)
```

### ③ 소비할 때 내는 세금 — 부가가치세(VAT)

우리가 물건을 살 때 이미 가격에 포함된 세금입니다.

```
커피 한 잔 5,000원
  → 실제 가격 4,545원 + VAT 455원(10%)
  → 가게 사장이 매 분기 국세청에 455원 납부
  → 소비자는 자신도 모르게 세금을 낸 것
```

**VAT 관련 일상 상식**

| 상황 | 세금 여부 |
|------|-----------|
| 마트·편의점·식당 구매 | 포함 (현금영수증 요청 시 소득공제 가능) |
| 대형 슈퍼마켓 농축산물 | **면세** (가공 전 식품) |
| 의료·교육 서비스 | **면세** |
| 해외직구 (150달러 초과) | 관세 + VAT 별도 부과 |
| 영수증 없이 현금 결제 | 소득공제 불가 → 현금영수증 반드시 챙길 것 |

### ④ 자동차 관련 세금

차를 사고 보유하고 운전할 때마다 세금이 발생합니다.

| 단계 | 세금 | 세율·금액 |
|------|------|-----------|
| **구입 시** | 취득세 | 차값의 7% (비영업용 승용차 기준) |
| **구입 시** | 개별소비세 | 출고가의 5% (2,000cc 초과) |
| **구입 시** | 교육세·부가세 | 개별소비세의 30%, 최종 합산가에 VAT 10% |
| **보유 시** | 자동차세 | 배기량 기준 (cc당 80~200원 × cc, 연 2회 납부) |
| **주유 시** | 교통·에너지·환경세 | 휘발유 1ℓ당 약 529원 고정세 포함 |

> 💡 **자동차세 연납 할인**: 1월에 1년치 선납하면 최대 5% 할인 혜택이 있습니다.

```
[예시] 2,000cc 국산 승용차 2,500만원 구입 시
  취득세: 2,500만원 × 7% = 175만원
  자동차세: 2,000cc × 200원 = 40만원/년
```

### ⑤ 부동산 관련 세금 — 사고·보유·팔 때

| 단계 | 세금 | 세율 |
|------|------|------|
| **살 때 (취득)** | 취득세 | 1주택 1~3%, 2주택 8%, 3주택 이상 12% |
| **보유 중** | 재산세 | 공시가격의 0.1~0.4% (매년 6·9월) |
| **보유 중** | 종합부동산세 (종부세) | 공시가 합산 9억원(1주택 12억) 초과 시 추가 부과 |
| **팔 때 (양도)** | 양도소득세 | 보유기간·주택수에 따라 6~82% |

**1세대 1주택 비과세 요건**

```
① 양도가액 12억원 이하
② 보유기간 2년 이상 (조정대상지역은 거주 2년 이상)
→ 위 조건 충족 시 양도소득세 전액 비과세
```

> 💡 **장기보유특별공제**: 3년 이상 보유 시 양도차익의 최대 80%까지 공제 (1주택 10년 이상 보유·거주 시).

### ⑥ 가족 간 돈 이체 — 증여세 주의

가족이라도 **일정 금액 이상 무상으로 돈·재산을 주면 증여세**가 발생합니다.

| 증여자 | 10년 합산 공제 한도 |
|--------|---------------------|
| 배우자 | **6억원** |
| 직계존속 (부모·조부모) | **5천만원** (미성년자 2천만원) |
| 직계비속 (자녀·손자녀) | **5천만원** |
| 기타 친족 (형제·자매 등) | **1천만원** |

```
[예시] 부모가 성인 자녀에게 8천만원 이체
  → 공제 5천만원 초과분 3천만원에 증여세 10% = 300만원 납부
  → 신고 기한: 증여일이 속하는 달의 말일부터 3개월 이내
```

> ⚠️ **주의**: 증여 사실을 숨기다 적발되면 가산세(무신고 20%, 납부지연 1일 0.022%)가 붙습니다. 금액이 크다면 반드시 신고하세요.

### ⑧ 상속세 기초 상식

부모님이 돌아가신 후 재산을 물려받을 때 부과됩니다.

| 과세표준 | 세율 | 누진공제 |
|----------|------|----------|
| 1억 이하 | 10% | — |
| 1억~5억 | 20% | 1천만원 |
| 5억~10억 | 30% | 6천만원 |
| 10억~30억 | 40% | 1억 6천만원 |
| 30억 초과 | 50% | 4억 6천만원 |

**기초공제·일괄공제**

```
일괄공제 5억원 (배우자 있을 경우 배우자 공제 최소 5억~최대 30억 추가)
→ 실질적으로 배우자+자녀 있는 일반 가정은 10억원까지 비과세인 경우 많음
신고 기한: 상속 개시일(사망일)로부터 6개월 이내
```

| 보고서 | 질문 | 항목 |
|--------|------|------|
| 재무상태표 (B/S) | 재산·빚이 얼마? | 자산 = 부채 + 자본 |

| 현금흐름표 (C/F) | 현금이 실제 얼마 들어왔나? | 영업·투자·재무 현금흐름 |

### ② 손익계산서 흐름

```
매출액
  − 매출원가
= 매출총이익
  − 판관비(SG&A)
= 영업이익(EBIT)
  ± 영업외손익(이자·환율)
= 세전이익(EBT)
  − 법인세
= 당기순이익
```

### ③ 세무조정 — 회계이익 vs 과세소득

```
회계이익(당기순이익)
  + 손금불산입(세법 불인정 비용)
  − 익금불산입(이중과세 방지)
= 과세표준 × 법인세율 = 산출세액
  − 세액공제·감면
= 납부 법인세
```

## F-A. 자산의 기본 개념

> 📺 [🎬 자산의 기본 개념](https://www.youtube.com/results?search_query=자산의+기본+개념+한국어)

> 📖 **Wikipedia**: [자산](https://ko.wikipedia.org/wiki/자산) · [재무상태표](https://ko.wikipedia.org/wiki/재무상태표) · [감가상각](https://ko.wikipedia.org/wiki/감가상각)

### ① 자산(資産)이란?

**자산(Asset)**은 과거 사건의 결과로 기업(또는 개인)이 **통제하고 있고, 미래에 경제적 이익이 유입될 것으로 기대되는 자원**입니다.

```
핵심 조건 (모두 충족해야 자산으로 인식):
  ① 통제(Control)    — 내가 사용하거나 처분할 수 있어야 함
  ② 경제적 효익      — 미래에 현금흐름이나 가치를 가져다줘야 함
  ③ 신뢰성 있는 측정 — 금액을 합리적으로 추정할 수 있어야 함

예시:
  ✅ 자산: 공장 건물, 현금, 매출채권, 특허권, 주식 보유
  ❌ 자산 아님: 계약 체결 전 예정 수주, 내부 개발 중인 브랜드 가치
```

### ② 자산의 분류 체계

**[분류 1] 유동성(환금성)에 따른 분류 — 재무상태표 기본 구조**

| 구분 | 기준 | 내용 | 예시 |
|------|------|------|------|
| **유동자산** (Current Assets) | **1년 이내** 현금화 가능 | 빠르게 쓰거나 팔 수 있는 자산 | 현금·예금, 매출채권, 재고자산, 단기금융상품 |
| **비유동자산** (Non-current Assets) | **1년 초과** 보유 | 장기 보유·운용 목적 자산 | 건물·기계, 토지, 투자주식, 특허권, 이연법인세 |

```
재무상태표(Balance Sheet) 왼쪽
─────────────────────────────────────────
자산 (Assets)
│
├── 유동자산
│   ├── 현금 및 현금성자산   100억
│   ├── 단기금융상품          50억
│   ├── 매출채권             200억
│   └── 재고자산             150억
│       유동자산 합계        500억
│
└── 비유동자산
    ├── 토지·건물·기계(유형)  800억
    ├── 특허권·영업권(무형)   100억
    └── 장기투자주식(투자)    200억
        비유동자산 합계     1,100억

자산 총계                   1,600억
```

|------|------|------|------|
| **유형자산** | Tangible Assets | 눈에 보이고 만질 수 있는 자산 | 토지, 건물, 기계, 차량, 설비 |

| **금융자산** | Financial Assets | 계약에 의한 청구권·수익권 | 현금, 예금, 주식, 채권, 파생상품 |

| 인플레이션 | 가격 상승 시 함께 오름 | 명목가치 고정 시 실질가치 하락 |
| 퀀트 활용 | DCF 밸류에이션 기반 | 가격·수익률 모델링 기반 |

### ③ 자산 = 부채 + 자본 (회계 등식)

회계의 가장 기본 원칙입니다. **자산은 어딘가에서 조달한 것**이며, 조달 원천이 부채(빌린 돈)와 자본(내 돈)입니다.

```
자산 (Assets) = 부채 (Liabilities) + 자본 (Equity)
─────────────────────────────────────────────────────
내가 가진 것 = 남한테 빌린 것 + 내 것

예시:
  집을 5억에 샀을 때:
    자산 5억 = 대출(부채) 3억 + 내 돈(자본) 2억

  회사가 자산 1,600억을 보유할 때:
    자산 1,600억 = 부채(차입금·채권 등) 900억 + 자본(주주 출자·유보이익) 700억
```

| 항목 | 정의 | 투자자에게 의미 |
|------|------|---------------|
| **자산(A)** | 기업이 보유한 모든 경제적 자원 | 기업이 뭘 갖고 있는가 |
| **부채(L)** | 타인에게 갚아야 할 의무 | 얼마를 빌렸는가 |
| **자본(E = A - L)** | 주주의 몫 (순자산) | 실질적인 주주 가치 |

> 💡 **순자산(Net Asset) = 자본 = 자산 - 부채**. 주가순자산비율(PBR)의 기준이 되는 값입니다. `PBR = 시가총액 ÷ 순자산`

### ④ 자산의 가치 측정 방법

같은 자산이라도 **어떤 기준으로 측정하느냐**에 따라 금액이 달라집니다.

| 측정 기준 | 영문 | 정의 | 특징 |
|---------|------|------|------|
| **취득원가** | Historical Cost | 처음 살 때 지불한 금액 | 안정적이나 현재 가치 반영 못함 |
| **공정가치** | Fair Value | 현재 시장에서 거래될 합리적 가격 | 현실적이나 변동성 있음 |
| **장부가치** | Book Value | 취득원가 - 누적 감가상각 - 손상차손 | 회계장부에 기록된 금액 |
| **시장가치** | Market Value | 실제 시장에서 거래되는 가격 | 주가·부동산 시세 등 |
| **청산가치** | Liquidation Value | 강제 매각 시 회수 가능 금액 | 장부가치보다 낮을 때가 많음 |

```python
def asset_values(acquisition_cost: float, useful_life: int, years_used: int,
                 market_price: float) -> dict:
    """유형자산 가치 비교 (정액법 감가상각 기준)"""
    annual_depreciation = acquisition_cost / useful_life
    accumulated_depreciation = annual_depreciation * years_used
    book_value = acquisition_cost - accumulated_depreciation
    unrealized_gain_loss = market_price - book_value  # 시장가치 - 장부가치

    return {
        "취득원가":          f"{acquisition_cost:,.0f}원",
        "연간 감가상각":      f"{annual_depreciation:,.0f}원",
        "누적 감가상각":      f"{accumulated_depreciation:,.0f}원",
        "장부가치 (Book)":    f"{book_value:,.0f}원",
        "시장가치 (Market)":  f"{market_price:,.0f}원",
        "미실현 평가손익":    f"{unrealized_gain_loss:+,.0f}원",
    }

# 예시: 10억짜리 건물, 내용연수 40년, 10년 사용, 현재 시장가 13억
result = asset_values(1_000_000_000, 40, 10, 1_300_000_000)
for k, v in result.items():
    print(f"  {k}: {v}")
```

```
  취득원가:          1,000,000,000원
  연간 감가상각:       25,000,000원
  누적 감가상각:      250,000,000원
  장부가치 (Book):    750,000,000원
  시장가치 (Market): 1,300,000,000원
  미실현 평가손익:   +550,000,000원  ← 장부가치보다 5.5억 높음
```

### ⑤ 감가상각 (Depreciation) — 자산 가치가 줄어드는 것을 인식하는 방법

**유형자산**은 사용할수록 노후화되므로, 그 가치 감소를 비용으로 인식합니다.

```
건물 10억, 내용연수 40년 (정액법)

  구입 시:   장부가치 10억
  10년 후:   장부가치 7.5억 (2.5억 감가)
  20년 후:   장부가치 5억   (5억 감가)
  40년 후:   장부가치 0원   (완전 상각)

매년 2,500만원씩 손익계산서에 '감가상각비' 비용 계상
→ 당기순이익 감소 BUT 실제 현금 유출은 없음 (비현금 비용)
```

| 감가상각 방법 | 원리 | 특성 |
|------------|------|------|
| **정액법** (Straight-line) | 매년 동일한 금액 상각 | 예측 쉬움, 국내 가장 일반적 |
| **정률법** (Declining Balance) | 장부가치의 일정 % 상각 → 초기에 더 많이 비용 인식 | 초기 이익 작고, 후기 이익 큼 |
| **생산량비례법** | 실제 생산량에 비례해 상각 | 기계 등 사용량 측정 가능한 자산에 적합 |

> 💡 **EBITDA**: 세전이익(E) + 이자(I) + 세금(T) + **감가상각(DA)**. 감가상각이 클수록 EBITDA와 순이익의 차이가 커집니다. 장치 산업(반도체·통신·항공)은 EBITDA로 수익성을 비교하는 이유가 여기에 있습니다.

## I. ISA — 개인종합자산관리계좌 (Individual Savings Account)

> 📺 [🎬 ISA 개인종합자산관리계좌](https://www.youtube.com/results?search_query=ISA+개인종합자산관리계좌+한국어)

> 📖 **Wikipedia**: [개인종합자산관리계좌](https://ko.wikipedia.org/wiki/개인종합자산관리계좌)

### ① ISA란?

**ISA(Individual Savings Account)**는 하나의 계좌 안에 예금·펀드·ETF·ELS·리츠 등 **다양한 금융상품을 담아 운용**하면서 **세금 혜택**을 받는 절세 투자 계좌입니다.

```
일반 계좌:
  펀드 A 이익 +100만원 → 세금 15.4만원
  펀드 B 손실 -50만원  → 손실 공제 없음
  실수령: 100 − 15.4 − 50 = 34.6만원

ISA 계좌:
  펀드 A 이익 +100만원
  펀드 B 손실 -50만원
  손익 통산 순이익 = 50만원
  비과세 한도 200만원 → 전액 비과세 (세금 0원!)
  실수령: 50만원
```

### ② ISA 유형·가입 조건·혜택 비교

| 구분 | 일반형 | 서민형 | 농어민형 |
|------|--------|--------|---------|
| 가입 대상 | 19세 이상 거주자 (근로·사업소득자) | 총급여 5천만원 또는 종합소득 3,800만원 이하 | 농어민 (소득 무관) |
| 비과세 한도 | **200만원** | **400만원** | **400만원** |
| 초과분 과세 | 9.9% **분리과세** | 9.9% 분리과세 | 9.9% 분리과세 |
| 의무 가입 기간 | **3년** | **3년** | **3년** |
| 연간 납입 한도 | **2,000만원** | 2,000만원 | 2,000만원 |
| 총 납입 한도 | **1억원** | 1억원 | 1억원 |

### ③ ISA 투자 가능 상품

| 상품 | 비고 |
|------|------|
| **예금·적금** | 확정 이자 |
| **국내 상장 ETF** | 주식·채권·리츠·원자재 ETF |
| **공모펀드** | 국내·해외 주식형·채권형 |
| **ELS·DLS** | 파생결합증권 |
| **리츠(REITs)** | 부동산투자신탁 |

> ⚠️ 해외 주식 직접 투자는 ISA에서 불가. ETF로 우회 투자해야 합니다.

### ④ ISA 세금 혜택 계산

```python
def isa_tax_benefit(profit: int, loss: int, account_type: str = "일반형") -> dict:
    net_profit = profit - loss
    tax_free_limit = 4_000_000 if account_type != "일반형" else 2_000_000
    if net_profit <= 0:
        return {"순이익": f"{net_profit:,}원", "세금": "0원"}
    taxable = max(0, net_profit - tax_free_limit)
    isa_tax = taxable * 0.099
    normal_tax = net_profit * 0.154
    return {
        "손익 통산 순이익": f"{net_profit:,}원",
        "비과세 한도": f"{tax_free_limit:,}원",
        "ISA 세금 (9.9%)": f"{isa_tax:,.0f}원",
        "일반 계좌 세금 (15.4%)": f"{normal_tax:,.0f}원",
        "절세 효과": f"{normal_tax - isa_tax:,.0f}원 절약",
    }

print("=== 일반형 ISA (이익 500만, 손실 100만) ===")
for k, v in isa_tax_benefit(5_000_000, 1_000_000, "일반형").items():
    print(f"  {k}: {v}")
```

### ⑤ ISA와 포트폴리오 연관성

```
계좌 유형별 자산 배분 최적화

  ISA 계좌         → 채권 ETF, 배당 ETF, ELS, 예금
                      (이자·배당 세금 절약 극대화)

  연금저축·IRP    → 장기 성장 자산 (주식형 ETF)
                      (납입 시 세액공제 + 과세 이연)

  일반 계좌       → 성장주, 해외 주식 직접 투자
                      (국내 주식 매매차익 비과세)
```

| 계좌 | 세금 혜택 | 의무 기간 | 적합 자산 |
|------|---------|---------|---------|
| **ISA** | 손익 통산 + 비과세 200/400만원 + 9.9% 분리과세 | 3년 | 채권 ETF, 배당 ETF |
| **연금저축** | 납입 세액공제 16.5%/13.2% + 과세 이연 | 55세 | 장기 주식 ETF |
| **IRP** | 연금저축 + 추가 세액공제 | 55세 | 위험자산 70% 제한 |
| **일반** | 없음 | 없음 | 해외 주식, 단기 매매 |

## J. 종합소득세 vs 금융소득종합과세 — 차이와 연계

> 📺 [🎬 종합소득세 vs 금융소득종합과세 차이와](https://www.youtube.com/results?search_query=종합소득세+vs+금융소득종합과세+차이와+연계+한국어)

### ① 종합소득세(종소세)란?

**종합소득세**는 개인이 1년 동안 **모든 소득을 합산**해서 납부하는 세금입니다.

합산되는 소득의 종류:

| 소득 유형 | 예시 |
|---------|------|
| **근로소득** | 월급, 보너스 |
| **사업소득** | 자영업·프리랜서 수입 |
| **이자소득** | 예금 이자, 채권 이자 |
| **배당소득** | 주식 배당금, 펀드 분배금 |
| **연금소득** | 국민연금, 사적 연금 |
| **기타소득** | 강의료, 원고료, 경품 등 |
| **임대소득** | 부동산 임대 수입 |

> 💡 **이자소득 + 배당소득 = 금융소득**. 이 금융소득이 연 **2,000만원을 초과**하면 종합과세 대상이 됩니다.

### ② 금융소득 분리과세 vs 종합과세

| 구분 | 분리과세 | 종합과세 |
|------|---------|---------|
| 대상 | 금융소득 연 2,000만원 **이하** | 금융소득 연 2,000만원 **초과분** |
| 세율 | **15.4%** 원천징수로 종결 | 다른 소득과 합산 → 최고 49.5% |
| 신고 | 원천징수로 끝 (별도 신고 불필요) | **5월 종합소득세 신고 필수** |
| 유리한 경우 | 금융소득 적을 때, 다른 소득 높을 때 | 다른 소득이 없거나 낮을 때 |

```
[금융소득 종합과세 예시]

연봉 8,000만원 직장인 (종합세율 35%)
이자+배당 3,000만원 수령 시:

  원천징수 이미 뗀 금액:
    3,000만원 × 15.4% = 462만원

  2,000만원 초과분 = 1,000만원 → 종합과세
  기존 소득 8,000만원 + 초과 금융소득 1,000만원
  = 9,000만원 → 세율 35% 적용

  추가 납부세액 = 1,000만원 × (35% - 15.4%) = 196만원 추가 납부

  → 고소득자일수록 금융소득이 많으면 세 부담 급증!
```

### ③ 금융소득 종합과세 회피 절세 전략

| 전략 | 방법 |
|------|------|
| **ISA 활용** | 이자·배당을 ISA 계좌 안에 담아 운용 → 금융소득 합산 제외 |
| **연금 전환** | 이자·배당 대신 연금 형태로 수령 → 연금소득 분리 과세 |
| **배우자 분산** | 금융자산을 배우자에게 증여해 각자 2,000만원 이하 유지 |
| **비과세 상품** | 장기저축성보험, 비과세 펀드, 농어촌특별세 면제 상품 |
| **해외 계좌 이용** | 해외 주식 매매차익은 양도소득(분류과세) → 금융소득 미포함 |

### ④ 과세·비과세·분리과세 개념 정리

| 구분 | 의미 | 예시 |
|------|------|------|
| **과세** | 세금을 내야 하는 소득 | 대부분의 이자·배당·근로소득 |
| **비과세** | 법에서 세금 면제 | ISA 비과세 한도 내, 장기저축성보험 만기 수령 |
| **분리과세** | 다른 소득과 합산하지 않고 낮은 세율로 끝냄 | ISA 초과분 9.9%, 금융소득 2천만원 이하 15.4% |
| **종합과세** | 다른 소득과 합산해 누진세율 적용 | 금융소득 2천만원 초과분 |
| **원천징수** | 지급자가 미리 세금을 떼고 지급 | 이자·배당 지급 시 증권사·은행이 15.4% 공제 |

```python
def financial_income_tax(
    financial_income: int,
    other_income: int,
    isa_income: int = 0
) -> dict:
    """
    financial_income: 이자+배당 합계 (원)
    other_income    : 근로·사업 등 기타 소득 합계
    isa_income      : ISA 계좌 내 이익 (금융소득 합산 제외)
    """
    # 구간별 소득세율 (지방소득세 포함)
    brackets = [
        (14_000_000,   0.066),
        (50_000_000,   0.165),
        (88_000_000,   0.264),
        (150_000_000,  0.385),
        (300_000_000,  0.418),
        (500_000_000,  0.44),
        (1_000_000_000, 0.462),
        (float('inf'), 0.495),
    ]
    def calc_tax(income):
        tax, prev = 0, 0
        for limit, rate in brackets:
            if income <= prev:
                break
            taxable = min(income, limit) - prev
            tax += taxable * rate
            prev = limit
        return tax

    # 금융소득 2,000만원 이하: 분리과세 15.4%로 종결
    threshold = 20_000_000
    withholding = min(financial_income, threshold) * 0.154
    excess = max(0, financial_income - threshold)

    if excess == 0:
        return {"분리과세 세금": f"{withholding:,.0f}원", "종합과세 추가": "없음"}

    # 초과분 종합과세
    total_income = other_income + excess
    gross_tax = calc_tax(total_income)
    base_tax = calc_tax(other_income)
    additional_tax = gross_tax - base_tax - withholding * (excess / financial_income)

    return {
        "금융소득": f"{financial_income:,}원",
        "ISA 제외 금액": f"{isa_income:,}원",
        "원천징수 (15.4%)": f"{withholding:,.0f}원",
        "종합과세 추가 납부": f"{max(0, additional_tax):,.0f}원",
    }

# 연봉 8천만원, 이자+배당 3천만원
result = financial_income_tax(30_000_000, 80_000_000)
for k, v in result.items():
    print(f"  {k}: {v}")
```

#### 🔑 상법 (商法) — 회사의 운영 규칙

- **쉽게 말하면?** 회사를 만들고, 운영하고, 없애는 모든 규칙을 담은 법이에요.

```
🏗️ 집 짓기 비유:
   건축법 없이 집을 지으면? → 마음대로 짓다가 무너짐
   상법 없이 회사 운영하면? → 사장 마음대로 해서 주주·직원 피해 발생
```

| 상법 주요 규정 | 내용 |
|-------------|------|
| **이사의 충실의무** | 이사는 회사와 주주를 위해 일해야 함 (개인 이익 먼저 X) |
| **이사의 선관주의의무** | 이사는 '평균적으로 주의 깊은 전문가'처럼 행동해야 함 |
| **주주총회 결의 사항** | 합병·분할·정관 변경·이사 선임은 주주 동의 필수 |
| **소수주주권** | 주식 1% 이상 보유 시 이사 해임 청구, 회계장부 열람 가능 |
| **배당 규제** | 이익 없으면 배당 불가, 채권자 보호 최우선 |

> 💡 **이사의 의무를 어기면?** 주주가 이사를 상대로 **주주대표소송**을 제기해 회사 손해를 배상받을 수 있어요.

#### 🔑 국세기본법 — 세금 절차의 기본 규칙

```
국세기본법은 "세금을 어떤 절차로 부과·징수·환급할지"를 정한 공통 규칙입니다.

쉬운 말로:
  소득세법·법인세법이 "무슨 세금을 내는지"를 정한다면,
  국세기본법은 "언제, 어떻게 내고 다투는지"를 정하는 절차법이에요.
```

| 핵심 항목 | 내용 | 투자자 관점 |
|---------|------|-----------|
| **신고·납부 기한** | 법정 신고기한 내 신고·납부 의무 | 해외주식 양도차익은 보통 5월 확정신고 |
| **가산세** | 무신고·과소신고·납부지연 시 추가 부담 | "늦게 내면 원금+추가비용" 구조 |
| **국세부과 제척기간** | 일반적으로 5년(사기·부정은 10년 등) | 과거 거래도 일정 기간 내 추징 가능 |
| **경정청구·불복** | 과다납부 시 환급 청구, 과세 불복 절차 보장 | 투자자 권리구제 통로 |

> 💡 핵심: "세금 종류"만 아는 것보다 **기한·증빙·신고 절차**를 지키는 것이 실제 절세와 리스크 관리에 더 중요합니다.

#### 🔑 소득세법 — 이자·배당·양도소득 과세의 본체

| 구분 | 과세 방식 | 기본 세율/구조 | 신고 포인트 |
|------|---------|--------------|-----------|
| **이자소득** | 원천징수 후 분리과세(원칙) | 14% + 지방소득세 1.4% = **15.4%** | 금융소득 합계 2,000만원 초과 시 종합과세 |
| **배당소득** | 원천징수 후 분리과세(원칙) | **15.4%** | 고배당·고금리 자산 비중 높을수록 종합과세 주의 |
| **국내 상장주식 양도소득** | 소액주주 원칙적 비과세 | 대주주·특정 비상장주식은 과세 | 대주주 기준·판정시점은 매년 최신 규정 확인 |
| **해외주식 양도소득** | 분류과세(별도 신고) | 연간 손익통산 후 기본공제 250만원, 초과분 **22%(지방세 포함)** | 다음 해 5월 확정신고·납부 |

```
투자자가 헷갈리는 핵심
  이자·배당 = "받을 때 원천징수"
  양도차익 = "연말 손익 계산 후 신고"

  즉, 돈 들어오는 시점의 과세 구조가 서로 다릅니다.
```

##### 납세의무의 성립(成立) 관점에서 보면

- **성립 시점(언제 세금 의무가 생기나?)**: 주식을 **매도 체결한 때(거래가 성사된 시점)** 납세의무가 성립합니다.
- **납부 방식(누가 실제로 내나?)**: 투자자가 따로 납부서를 작성하기보다, 보통 **증권사가 매도대금에서 자동 공제**해 원천적으로 처리합니다.

```
초등학생 비유
  중고 장난감을 친구에게 "팔기로 약속이 딱 된 순간"
  → "수수료를 내야 하는 약속"이 바로 생김(= 납세의무 성립)
  실제 돈 전달할 때는
  → 엄마가 수수료를 먼저 떼고 남은 돈을 주는 것과 비슷함(= 원천 공제)
```

#### 🔑 양도소득세 (Capital Gains Tax)

```
🏠 부동산을 팔 때는 보통 양도소득세가 과세되고,
📈 금융투자에서는 자산 종류·거주자 여부·시장(국내/해외)에 따라 과세 방식이 달라집니다.

🌍 해외 주식 양도소득세(대표 구조):
   1년 손익통산 후 순이익 - 250만원 기본공제
   × 22% (지방소득세 포함) = 납부세액

   예시: 해외 주식 이익 1,000만원
   → (1,000만 - 250만) × 22% = 165만원 세금
   → 다음 해 5월에 직접 신고·납부
```

#### 법인세법 — 회사가 내는 세금의 규칙

```
법인세법 = 회사(법인)가 언제, 얼마나 세금을 내야 하는지 정한 법

핵심 개념:
   익금(益金) = 세법상 수익 (회계상 수익 + 추가 과세항목)
   손금(損金) = 세법상 비용 (회계상 비용 중 세법이 인정하는 것)
   과세표준 = 익금 - 손금
   납부세액 = 과세표준 × 세율 - 세액공제
```

| 법인세법 중요 규정 | 쉬운 설명 |
|----------------|---------|
| **접대비 한도** | 밥값·선물비용은 일정 한도 내에서만 비용 인정 |
| **감가상각 규정** | 건물·기계는 정해진 방식대로만 비용 처리 가능 |
| **이월결손금** | 올해 손실을 최대 15년간 다음 이익에서 차감 가능 |
| **세액공제·감면** | R&D 투자, 설비 투자, 고용 창출 시 세금 감면 혜택 |

```python
# 법인세 세액공제 간단 예시
# 연구개발(R&D) 비용 세액공제 (중소기업 최대 25%)
rd_expense = 100_000_000  # 1억원 R&D 투자
tax_credit_rate = 0.25    # 중소기업 25%
tax_credit = rd_expense * tax_credit_rate

corporate_tax_before = 50_000_000  # 산출세액 5천만원
corporate_tax_after  = corporate_tax_before - tax_credit

print(f"R&D 투자액:    {rd_expense:>12,}원")
print(f"세액공제:      {tax_credit:>12,}원")
print(f"납부 법인세:   {max(0, corporate_tax_after):>12,}원")
# → 납부 법인세: 25,000,000원 (세금 절반으로 줄어듦!)
```

### ⑦ 법인 설립·운영 단계별 법적 절차 📋

```
[법인 설립 ~ 운영 흐름도]

① 정관 작성
   └── 회사 이름, 목적, 자본금, 주식 수 등 기재

② 자본금 납입
   └── 주주들이 주식 대금을 은행에 입금
   └── 최저 자본금 제한 없음 (단, 실무상 최소 100만원 이상)

③ 법인 등기 (법원 등기소)
   └── 법인 탄생! 이때부터 법적 인격 생김

④ 사업자 등록 (세무서)
   └── 세금 납부·사업 가능

⑤ 4대보험 취득 신고 (직원 고용 시)
   └── 국민연금·건강보험·고용보험·산재보험

⑥ 부가세·법인세 신고 (정기)
   └── 부가세: 연 2회 (1월, 7월)
   └── 법인세: 연 1회 (결산월 + 3개월 내)

⑦ 외부감사 (해당 시)
   └── 자산 120억 이상 등 일정 규모 이상이면 필수

⑧ 주주총회 (연 1회 이상)
   └── 재무제표 승인, 이사 선임, 배당 결정
```

```python
# 법인 설립 비용 간단 추정
def corporation_setup_cost(capital: int) -> dict:
    """
    주식회사 설립 시 초기 비용 추정 (2024년 기준)
    capital: 자본금 (원)
    """
    # 등록면허세: 자본금 × 0.4% (서울시 기준, 중과세율 3배 적용 시)
    registration_tax = capital * 0.004
    # 교육세: 등록면허세 × 20%
    education_tax = registration_tax * 0.2
    # 공증료 (공증인 방문 기준 대략)
    notarization_fee = 300_000
    # 법무사 수수료 (대략)
    agent_fee = 400_000

    total = registration_tax + education_tax + notarization_fee + agent_fee
    return {
        "자본금":           f"{capital:,}원",
        "등록면허세":       f"{registration_tax:,.0f}원",
        "교육세":           f"{education_tax:,.0f}원",
        "공증료(추정)":     f"{notarization_fee:,}원",
        "법무사 수수료(추정)": f"{agent_fee:,}원",
        "총 설립 비용(추정)":  f"{total:,.0f}원",
    }

for cap in [10_000_000, 50_000_000, 100_000_000]:
    result = corporation_setup_cost(cap)
    print(f"\n[자본금 {cap:,}원 설립 시]")
    for k, v in result.items():
        print(f"  {k}: {v}")
```

#### 📄 법인등기부 (法人登記簿) — 회사의 '주민등록증'

```
법인등기부등본 = 법원 등기소가 관리하는 회사의 공식 신분증

  ┌─────────────────────────────────────────────────────┐
  │ 상호(회사명)          │ (주)퀀트랩                       │

  │ 설립연월일            │ 2022년 03월 15일                 │
  │ 목적(사업 목적)       │ 소프트웨어 개발업, 투자 자문업 등  │
  │ 1주의 금액            │ 5,000원                          │
```

**등기부등본 열람·발급 방법**

```
방법 1 — 온라인 (무료 열람 / 유료 발급)

   → 법인등기 → 법인명 또는 등록번호 검색

방법 2 — 방문 발급
   → 가까운 지방법원·등기소 민원실
   → 용지에 회사 상호·등록번호 기재 후 창구 신청

💡 계약서 체결, 금융 거래, 정부 지원 신청 시
```

#### 🧾 법인세 신고 실무 — 연간 세금 일정과 절차

```
법인세 납부 일정 (12월 결산 법인 기준)

  1월       2월       3월       4월 ← 신고·납부 마감 (3월 31일)
  │         │         │         │
  결산 준비  재무제표    세무조정   법인세 신고·납부
  시작       확정       작업      (전년도 소득 기준)

  8월 ← 중간예납 마감 (8월 31일)
  │
  직전 사업연도 법인세액의 50% 또는 실적 기준 선납
```

**법인세 신고 5단계 흐름**

```
Step 1 결산 (決算)
   → 1년치 매출·비용·이익을 계산해 재무제표 완성
   → 손익계산서·대차대조표·현금흐름표 확정

Step 2 세무조정 (稅務調整)
   → 회계상 이익 ≠ 세법상 과세소득
   → 익금산입·손금산입·익금불산입·손금불산입 항목 조정

       회계상 당기순이익
       + 익금산입 (세법이 수익으로 추가하는 항목)
       - 손금산입 (세법이 비용으로 추가 인정하는 항목)
       ±세무조정
       = 각사업연도 소득금액 (과세표준)

Step 3 과세표준·세액 계산
   → 이월결손금 차감 → 비과세소득 공제 → 세율 적용
   → 세액공제·감면 적용 → 최종 납부 세액 산출

Step 4 법인세 신고서 제출
   → 홈택스(www.hometax.go.kr) 전자신고 또는 세무사 대리
   → 사업연도 종료일로부터 3개월 이내 (12월 결산: 익년 3월 31일)

Step 5 법인세 납부
   → 신고서 제출 기한과 동일
   → 분납 가능: 납부세액 1,000만원 초과 시 2개월 내 분할 납부
```

**법인세 세율 (2024년 기준)**

| 과세표준 | 세율 |
|---------|------|
| 2억원 이하 | 9% |
| 2억원 초과 ~ 200억원 이하 | 19% |
| 200억원 초과 ~ 3,000억원 이하 | 21% |
| 3,000억원 초과 | 24% |

```python
# 법인세 신고 시 중간예납 계산 예시
def corporate_tax_midyear_prepayment(prior_year_tax: int) -> dict:
    """
    중간예납액 계산 (직전 사업연도 세액 기준)
    prior_year_tax: 직전 사업연도 법인세 납부액 (원)
    납부 기한: 사업연도 개시 후 6개월이 되는 날로부터 2개월 이내
    (12월 결산 법인 기준: 매년 8월 31일)
    """
    prepayment = prior_year_tax * 0.5   # 50% 선납
    return {
        "직전연도 납부 법인세":  f"{prior_year_tax:,}원",
        "중간예납액 (50%)":     f"{prepayment:,.0f}원",
        "납부 기한":            "8월 31일",
        "신고 방식":            "홈택스 전자신고 또는 세무사 대리",
    }

example = corporate_tax_midyear_prepayment(20_000_000)
for k, v in example.items():
    print(f"  {k}: {v}")
# → 직전연도 납부 법인세: 20,000,000원
# → 중간예납액 (50%): 10,000,000원
# → 납부 기한: 8월 31일
```

#### 📚 기장 (記帳) — 매일매일의 재무 기록

```
기장(記帳) = 회사의 모든 돈 거래를 장부에 기록하는 것

🍱 도시락 가게 비유:
   재료비 10,000원 → 장부에 '비용 10,000원' 기록
   도시락 판매 15,000원 → 장부에 '매출 15,000원' 기록
   → 이렇게 매일 기록 = 기장!
```

**복식부기 의무 — 법인은 예외 없이 필수**

```
단식부기 (개인 가계부 방식)      복식부기 (법인 의무)
───────────────────────       ───────────────────────────────
수입 15,000     지출 10,000    차변(借邊)        대변(貸邊)
                              매출채권 15,000 / 매출 15,000
                              매입원가 10,000 / 현금 10,000

→ 단순하지만 자산·부채 파악 어려움   → 자산·부채·자본 완전 파악 가능
→ 법인은 사용 불가 (세법상 무기장)   → 법인세법 제112조: 법인 복식부기 의무
```

**기장 월간 사이클**

```
매월 반복되는 기장 업무 흐름

  1~10일  ── 전월 거래내역 정리
              └── 세금계산서·영수증 수집·분류
              └── 통장·카드 내역 대사(對査)

  10~20일 ── 분개(分介) 및 전표 입력
              └── 회계 프로그램(더존·세무사랑 등)에 입력
              └── 매입·매출 전표 구분

  20~25일 ── 부가세 예정 확인 (분기 해당 월)
              └── 매출세액 - 매입세액 = 납부(환급)세액 계산

  말일    ── 월계표·시산표 점검
              └── 오류 항목 수정
              └── 다음달 업무 인계
```

| 기장 방식 | 설명 | 세금 혜택 |
|----------|------|---------|
| **성실 기장** | 모든 거래를 정확히 기록 | 세액공제·감면 최대 적용 가능 |
| **무기장** | 장부를 아예 기록하지 않음 | 산출세액의 **20% 가산세** 부과 |
| **불성실 기장** | 기록은 했으나 허위·누락 | 부정 과소신고 가산세 40% 부과 |

> 💡 법인은 무기장이 허용되지 않습니다. 반드시 복식부기로 기장해야 하며,  
> 미기장 시 세금 불이익 외에도 세무조사 선정 위험이 높아집니다.

### ⑪ 법인 대출 심사 기준 및 대출금 운용 범위 💳

> 법인이 은행·금융기관에서 대출을 받으려면 **개인 대출과는 다른 심사 기준**이 적용됩니다.  
> 또한 대출금으로 무엇을 해도 되는지(주식 투자·부동산 임대 등) 자주 질문이 나옵니다.  
> 여기서는 심사 기준과 운용 범위를 실무 관점에서 정리합니다.

---

#### 📋 1. 법인 대출 심사 기준

은행은 법인 대출 신청 시 **재무 요인**과 **비재무 요인** 두 축으로 심사합니다.

##### ① 재무(定量) 심사 항목

| 심사 항목 | 주요 지표 | 기준 예시 (은행마다 상이) |
|---------|---------|----------------------|
| **매출·영업이익** | 매출액 증가율, 영업이익률 | 최근 3년 흑자 지속 여부 |
| **부채비율** | 총부채 ÷ 자기자본 × 100 | 200% 이하 (업종 평균 대비) |
| **유동비율** | 유동자산 ÷ 유동부채 × 100 | 100% 이상 (단기 지급 능력) |
| **이자보상배율** | 영업이익 ÷ 이자비용 | 1.5배 이상 (이자 상환 여력) |
| **자기자본비율** | 자기자본 ÷ 총자산 × 100 | 30% 이상 권장 |
| **현금흐름** | 영업활동 현금흐름 (FCF) | 지속적 플러스 여부 |
| **담보·보증** | 부동산·매출채권·동산 담보, 대표이사 연대보증 여부 | 담보 인정비율 60~80% |

```
📊 부채비율 계산 예시:
   자기자본 5억원, 총부채 8억원
   → 부채비율 = 8억 ÷ 5억 × 100 = 160%  ← 양호
   → 200%를 넘으면 고위험으로 금리 가산 또는 한도 축소
```

##### ② 비재무(定性) 심사 항목

| 심사 항목 | 세부 내용 |
|---------|---------|
| **업력·사업 안정성** | 창업 후 2년 이상, 동종업계 경쟁력 |
| **대표이사 신용도** | 개인 신용점수, 과거 연체·부도 이력 |
| **법인 신용등급** | NICE·KCB 등 CB사 법인 신용등급 |
| **사업 목적 부합성** | 대출 자금 용도가 사업 목적과 연관 있는지 |
| **업종 리스크** | 규제 산업(도박·유흥 등)은 대출 제한 |
| **결산 재무제표 신뢰성** | 외부 감사 여부, 세무조정계산서 일치 여부 |

> 💡 **실무 팁**: 창업 초기 법인은 재무 이력이 짧아 **대표이사 신용·담보**가 사실상 핵심 심사 요소가 됩니다.  
> 일정 규모가 되면 법인 자체의 신용등급·재무비율이 중심이 됩니다.

##### ③ 법인 대출 종류별 특징

| 대출 종류 | 설명 | 주요 용도 |
|---------|-----|---------|
| **운전자금 대출** | 원자재 구매·인건비·임대료 등 일상 영업비용 충당 | 매출 발생 전 현금 부족 시 |
| **시설자금 대출** | 공장·설비·사무실 등 고정자산 취득 | 설비 투자, 사무실 매입 |
| **무역금융** | 수출입 결제 대금 일시 지원 | 수출기업 L/C 개설 |
| **기업어음(CP) 할인** | 법인이 발행한 어음을 은행이 할인 매입 | 단기 유동성 확보 |
| **프로젝트 파이낸싱(PF)** | 특정 프로젝트의 미래 현금흐름 담보 | 부동산 개발, 대규모 인프라 |

##### ④ 보증기관(신보·기보 등) 종류와 역할

담보가 부족한 중소·중견 법인은 보증기관 보증서를 통해 은행 대출 승인 가능성을 높일 수 있습니다.

| 구분 | 기관 | 핵심 역할 | 주로 적합한 기업 |
|------|------|----------|------------------|
| **신용보증** | **신용보증기금(KODIT)** | 재무·신용 기반 보증서 발급, 일반 운전자금/시설자금 보증 | 업력 있는 중소·중견기업, 일반 제조·서비스업 |
| **기술보증** | **기술보증기금(KIBO)** | 기술평가(TCB) 기반 보증서 발급, R&D·기술사업화 자금 지원 | 특허·기술력 중심 스타트업/혁신기업 |
| **지역신보** | **지역신용보증재단** | 소상공인·지역 중소기업 보증, 지방자치단체 연계 특례보증 | 소상공인·영세 법인, 지역 기반 사업자 |
| **민간보증** | **서울보증보험(SGI)** | 이행보증·지급보증 등 상거래/계약 관련 보증 제공 | 계약이행·납품·입찰 보증이 필요한 기업 |

> 💡 **실무 포인트**: 은행은 "차주 신용" + "보증기관 보증서"를 함께 심사합니다.  
> 보증비율·보증료율·보증기간은 기관/상품별로 다르며, 보증 심사 승인 후 은행 대출 실행으로 연결됩니다.

##### ⑤ 법인 대출 이자율 구조(실무)

법인 대출 금리는 통상 아래 구조로 결정됩니다.

참고: **COFIX(Cost of Funds Index)**는 은행권의 실제 평균 자금조달비용을 반영한 지수로, 변동금리 대출의 기준금리 산정에 자주 활용됩니다.

```
대출금리 = 기준금리(시장금리 또는 COFIX 등) + 가산금리(신용·담보·업종 리스크) - 우대금리
```

| 구분 | 금리 수준 예시(변동 가능) | 특징 |
|------|------------------------|------|
| **담보대출(우량 법인)** | 연 4~7% 내외 | 담보가치·재무안정성이 좋을수록 금리 인하 가능 |
| **신용대출(일반 중소법인)** | 연 6~10% 내외 | 재무비율·대표이사 신용·업황에 따라 가산금리 확대 |
| **보증부대출(신보/기보 연계)** | 연 4.5~8.5% 내외 + 보증료(연 0.5~2.0% 내외) | 담보 부족 기업 접근성 개선, 보증료 포함한 총조달비용 확인 필요 |
| **정책자금 연계대출** | 일반 시중금리 대비 상대적 저금리 가능 | 정책 목적(창업·수출·고용)에 따라 지원조건 상이 |

> ⚠️ **주의**: 실제 금리는 기준금리 변동, 회사 신용등급, 담보, 보증 여부, 은행 내부등급, 우대조건 충족 여부에 따라 달라집니다.  
> 따라서 계약 전 반드시 **약정금리 + 보증료 + 부대비용**을 합산해 비교해야 합니다.
> 예시: 대출금리 연 5.8% + 보증료율 연 1.2%라면, 단순 총조달비용은 연 7.0%(부대비용 별도) 수준으로 볼 수 있습니다.

##### ⑥ 법인 대출 절차(신청~사후관리)

| 단계 | 주요 내용 | 실무 체크포인트 |
|------|-----------|----------------|
| **1. 사전 상담** | 자금 용도(운전/시설), 필요 금액·기간 확인 | 용도와 실제 집행 계획을 먼저 문서화 |
| **2. 서류 접수** | 재무제표, 부가세 과세표준증명, 국세·지방세 납세증명, 사업계획서 등 제출 | 누락 서류가 있으면 심사 지연 |
| **3. 신용·재무 심사** | 은행 내부등급, 현금흐름, 담보가치, 업종리스크 평가 | 최근 3개년 재무자료 정합성 중요 |
| **4. 보증 심사(필요 시)** | 신보·기보·지역신보 보증 신청 및 승인 | 보증한도·보증비율·보증료율 확인 |
| **5. 약정·실행** | 대출약정 체결, 금리·상환조건 확정 후 실행 | 중도상환수수료·기한이익상실 조항 확인 |
| **6. 사후 관리** | 자금 용도 점검, 재무상태 모니터링, 만기 연장 심사 | 용도 외 사용 시 즉시상환·여신제한 위험 |

#### 📈 2. 법인 대출금으로 주식투자 가능한가?

**결론 먼저**: 원칙적으로 **은행·금융기관 대출금으로 주식 투자는 금지 또는 강력히 제한**됩니다.

##### ① 규제 근거

| 규제 | 내용 |
|-----|-----|
| **은행 여신규정** | 운전자금·시설자금 대출금은 해당 용도에만 사용해야 함. 타 용도 유용 시 기한이익 상실(즉시 상환) 조항 적용 |
| **여신거래약관** | 대출 계약서에 "대출금을 유가증권 취득, 투기성 자산 매입에 사용할 수 없다"는 조항 명시 |
| **금융감독원 지도** | 부동산·주식 등 투기성 자금으로 흘러가는 것을 방지하기 위한 용도 외 사용 모니터링 |

```
⚠️ 기한이익 상실 (期限利益 喪失):
   = "분할 상환하기로 한 권리를 잃고 전액 즉시 갚아야 하는 상황"
   
   예: 1억원을 3년 만기로 빌렸는데 주식 투자에 사용하다 발각
     → 은행이 잔액 전액을 즉시 갚으라고 요구할 수 있음
```

##### ② 예외적으로 허용되는 경우

| 허용 유형 | 조건 |
|---------|-----|
| **기업 M&A·전략적 투자** | 사업 목적과 직접 연관된 타 법인 주식 취득 (지분 투자, 자회사 편입) → 사전에 은행에 용도 명시 후 시설자금 등으로 별도 약정 |
| **벤처·신기술 사업자 투자** | 중소기업창업투자회사·신기술금융사 등 정책 금융 지원 시 투자 목적 명시 허용 |
| **자기자금(자본금·유보이익)** | 대출금이 아닌 법인 고유 자금은 정관·이사회 결의 범위 내에서 주식 투자 가능 |

> ⚠️ **주의**: 대출금을 주식 계좌로 이체한 뒤 투자하는 행위는 금융기관이 이상 거래로 탐지할 수 있습니다.  
> 적발 시 대출 기한이익 상실, 향후 여신 제한, 세무조사 연계 위험이 있습니다.

#### 🏠 3. 법인 대출금으로 부동산 임대 가능한가?

**결론 먼저**: **운전자금·시설자금 명목 대출금으로 임대용 부동산 취득은 원칙적으로 불가**합니다.  
단, 법인이 부동산 임대업을 사업 목적으로 등록하고 적절한 대출 상품을 이용하는 경우는 다릅니다.

##### ① 불가한 경우 (용도 외 사용)

```
예: 제조업 법인이 "운전자금" 명목으로 빌린 3억원으로
    수익형 오피스텔을 구입해 임대 수익을 얻으려는 경우
    
→ 대출 계약서상 용도 위반
→ 발각 시 기한이익 상실 + 향후 여신 제한
→ 세무서에서 업종 외 수입으로 법인세 추가 과세
```

##### ② 가능한 경우 (적법한 구조)

| 구조 | 조건 |
|-----|-----|
| **법인 사업목적에 부동산 임대업 포함** | 정관에 '부동산 임대업' 추가, 사업자등록에 업종 코드 등록 |
| **부동산 담보 시설자금 대출** | 임대용 건물 취득 목적임을 명시하고 부동산 담보 대출 실행 |
| **PF(프로젝트 파이낸싱)** | 개발 후 임대 목적의 건물 신축 시 PF 활용 가능 |
| **기업 업무용 부동산** | 사무실·공장 등 자가 사용 목적 부동산 취득은 시설자금으로 정상 처리 |

> 💡 **실무 조언**: 부동산 임대를 목적으로 한다면 처음부터 정관에 업종을 넣고, 은행에 용도를 투명하게 신고하는 것이 리스크를 없애는 방법입니다.

##### ③ 법인의 부동산 임대 시 세금 처리

| 항목 | 내용 |
|-----|-----|
| **임대 수입 과세** | 법인세 과세 대상 익금으로 처리 |
| **부가가치세** | 상가 임대는 부가세 과세, 주택 임대는 면세 |
| **취득세** | 부동산 취득 시 취득세 납부 (일반 법인 4.4%, 주택 중과 가능) |
| **종합부동산세** | 법인 보유 주택은 중과세 (2주택 이상 시 최고 6%) |

#### 💼 4. 법인 대출금으로 허용되는 자산운용 범위

##### ① 대출금 용도별 허용·제한 정리

| 자산운용 항목 | 운전자금 대출 | 시설자금 대출 | 자기자금(자본금·유보이익) |
|------------|:---------:|:---------:|:-------------------:|
| 원자재·재고 구입 | ✅ 가능 | ❌ 불가 | ✅ 가능 |
| 인건비·임대료 지급 | ✅ 가능 | ❌ 불가 | ✅ 가능 |
| 공장·설비 취득 | ❌ 불가 | ✅ 가능 | ✅ 가능 |
| 사무실·업무용 부동산 취득 | ❌ 불가 | ✅ 가능 | ✅ 가능 |
| 임대용 부동산 취득 | ❌ 불가 | ❌ 불가 (별도 약정 필요) | ✅ 가능 (사업목적 등록 시) |
| 상장주식·ETF 투자 | ❌ 불가 | ❌ 불가 | ⚠️ 가능하나 이사회 결의 필요 |
| 타 법인 지분 취득 (M&A) | ❌ 불가 | ❌ 불가 (별도 M&A 금융) | ✅ 가능 (이사회·주주총회 결의) |
| 금·원자재 등 실물자산 | ❌ 불가 | ❌ 불가 | ⚠️ 사업 관련성 판단 필요 |
| 암호화폐(가상자산) | ❌ 불가 | ❌ 불가 | ⚠️ 정관·이사회 결의·회계처리 주의 |
| 예금·MMF 단기 운용 | ✅ 일시적 가능 | ✅ 일시적 가능 | ✅ 가능 |

> ⚠️ 표의 "자기자금 ✅"는 법률상 금지가 없다는 의미이며, 실제로는 **이사회 결의**, **정관 목적 사업 범위**, **세무처리** 등을 반드시 확인해야 합니다.

##### ② 법인이 자기자금으로 금융자산에 투자할 때 주의사항

```
📌 법인 금융투자 체크리스트:

1️⃣ 정관 확인
   → 정관 목적 사업에 "금융상품 투자", "유가증권 취득" 등이 포함되어 있는지

2️⃣ 이사회(또는 주주총회) 결의
   → 주요 투자 결정은 이사회 결의가 필요 (상법 393조)
   → 자기자본 10% 이상의 대규모 투자는 주주총회 특별결의 요구 가능

3️⃣ 회계처리
   → 단기 투자: 당기손익-공정가치 측정 금융자산 (FVPL)
   → 장기 투자·전략 투자: 지분법 적용 관계기업 또는 FVOCI
   → 손익이 법인세 과세 대상

4️⃣ 손실 리스크
   → 투자 손실은 법인 재무제표에 그대로 반영 → 대출 심사에 불리해짐
   → 금융기관이 재무비율 악화 감지 시 조기 상환 요구 가능
```

##### ③ 법인 대출금 운용 원칙 요약

```
[법인 대출금 운용 원칙]

✅ 허용:   대출 계약서에 명시된 용도 (운전자금·시설자금)
✅ 허용:   단기 여유자금의 예금·MMF 운용 (용도 내 일시 운용)
⚠️ 조건부: 사업 목적 연관된 법인 지분 취득 (사전 은행 협의 필수)
❌ 금지:   주식·코인·임대용 부동산 등 투기성 자산 취득
❌ 금지:   계열사 불법 자금 지원 (공정거래법 내부거래 규제)

위반 시 결과:
  1단계 → 기한이익 상실 (전액 즉시 상환 요구)
  2단계 → 신용등급 하락, 금융기관 여신 제한
  3단계 → 세무조사 연계 (용도 외 사용 = 가지급금 처리 가능)
  4단계 → 대표이사 형사 책임 (사기적 대출, 배임 등)
```

| 위반 리스크 | 설명 |
|-----------|-----|
| **기한이익 상실** | 잔액 전액 즉시 상환 요구 |
| **가지급금 처리** | 세무서에서 대표자 가지급금으로 간주 → 인정이자 과세 |
| **배임죄** | 법인 자금을 대표이사가 사적으로 운용 시 형사책임 (형법 355조) |
| **금융거래 제한** | 여신 제한·블랙리스트 등록 → 향후 대출 불가 |

#### 🗺️ 법인 대출 & 자산운용 한눈에 정리

```
[법인 자금 조달 및 운용 구조]

법인 자금 조달
├── 자기자금 (자본금·유보이익)
│     └── 운용 자유도 높음 (정관·이사회 범위 내)
└── 외부 차입 (은행 대출)
      ├── 운전자금 대출 → 영업비용(원자재·인건비 등)에만 사용
      └── 시설자금 대출 → 고정자산(설비·사무실) 취득에만 사용

주식 투자 (대출금 사용)       → ❌ 원칙 금지
부동산 임대 (대출금 사용)     → ❌ 원칙 금지 (별도 약정 필요)
예금·MMF 단기 운용 (대출금)  → ✅ 일시적 허용
M&A·전략 지분투자 (대출금)   → ⚠️ 사전 협의·별도 약정 후 허용

주식 투자 (자기자금)          → ✅ 이사회 결의 후 가능
부동산 임대 (자기자금)        → ✅ 사업목적 등록 후 가능
```

> 💡 **핵심 원칙**: 대출금은 "신청한 용도 그대로" 사용해야 합니다. 투자·임대 등 수익 창출 목적의 자산운용은 **자기자금**으로, 또는 **전용 금융 상품**(부동산 담보 대출, M&A 금융 등)을 이용하는 것이 원칙입니다.

## H. 퀀트 분석에서 세금이 중요한 이유

> 📺 [🎬 퀀트 분석에서 세금이 중요한](https://www.youtube.com/results?search_query=퀀트+분석에서+세금이+중요한+이유+한국어)

| 분석 항목 | 세금 연관성 |
|-----------|-----------|

```python
print(f"실효 법인세율: {etr*100:.1f}%")   # 20.0%
print(f"명목 법인세율: 24.0% → {(0.24-etr)*100:.1f}%p 절세 효과")
```

### A. 업종과 업태 — 기업을 분류하는 두 가지 축

투자 분석에서 기업을 분류할 때 **업종**과 **업태** 두 개념이 자주 혼용됩니다. 엄밀히 구분하면 다음과 같습니다.

#### ① 업종 (業種, Industry / Sector)

**"무엇을 만들거나 서비스하는가"** — 취급하는 재화·서비스의 종류로 분류합니다.

| 업종 | 대표 기업 | 비고 |
|------|---------|------|
| 반도체 | 삼성전자, SK하이닉스 | 제조업 내 세부 업종 |
| 자동차 | 현대차, 기아 | 제조업 |
| 금융(은행) | KB국민, 신한 | 금융업 |
| 유통·소매 | 이마트, 쿠팡 | 서비스업 |
| 바이오·제약 | 셀트리온, 한미약품 | 제조+서비스 혼합 |

> 주식시장에서 업종은 **KOSPI 업종 분류(24종)**나 **GICS(글로벌산업분류기준)** 로 표준화됩니다.

#### ② 업태 (業態, Business Format / Type of Business)

**"어떤 방식으로 사업을 하는가"** — 영업 형태·유통 방식·비즈니스 모델로 분류합니다.

| 업태 | 설명 | 예시 |
|------|------|------|
| 제조업 | 원재료를 가공해 완성품 생산 | 삼성전자(가전), 현대차(자동차) |
| 도매업 | 대량 매입해 소매상에 공급 | 의류 도매시장 |
| 소매업 | 최종 소비자에게 직접 판매 | 편의점, 백화점, 쿠팡 |
| 서비스업 | 무형의 서비스 제공 | 병원, 학원, 컨설팅 |
| 플랫폼업 | 공급자-소비자 연결 중개 | 카카오, 배달의민족, 에어비앤비 |
| 구독형 | 월정액 반복 과금 | 넷플릭스, 어도비 |
| 프랜차이즈 | 브랜드·시스템을 가맹점에 제공 | 맥도날드, 스타벅스 |

#### ③ 업종 vs 업태 — 핵심 차이

```
업종: "무엇을 팔아?"        업태: "어떻게 팔아?"
─────────────────────────────────────────────
같은 업종(식품)이라도:
  CJ제일제당 → 제조업 (공장에서 만들어 납품)
  이마트 → 소매업 (소비자에게 직접 판매)
  마켓컬리 → 플랫폼·소매업 (온라인 새벽배송)
```

> 💡 **투자 분석 활용**: 같은 업종 내에서도 업태가 다르면 수익 구조(매출원가율, 재고 위험, 마진)가 전혀 다릅니다. 제조업체와 플랫폼 기업의 PER를 단순 비교하면 안 되는 이유입니다.

### B. 산업 분류 체계 — GICS와 KSIC

#### ① GICS (Global Industry Classification Standard)

MSCI와 S&P가 공동 개발한 **글로벌 표준 산업 분류**. 글로벌 ETF·지수·포트폴리오 분석의 공통 언어입니다.

```
GICS 4단계 분류

섹터 (11개)
  └── 산업군 (25개)
        └── 산업 (74개)
              └── 세부산업 (163개)
```

| 섹터 | 대표 기업 | 특징 |
|------|---------|------|
| **에너지** | 엑슨모빌, SK이노베이션 | 유가에 민감 |
| **소재** | POSCO, 다우케미칼 | 경기 민감 |
| **산업재** | 현대건설, 캐터필러 | 설비투자 연동 |
| **경기소비재** | 현대차, 아마존 | 경기에 따라 소비 변화 |
| **필수소비재** | CJ제일제당, P&G | 경기 방어적 |
| **헬스케어** | 삼성바이오, 존슨앤존슨 | 고령화 수혜 |
| **금융** | KB금융, JP모건 | 금리에 민감 |
| **IT** | 삼성전자, 애플, NVIDIA | 성장주 대표 |
| **통신서비스** | SK텔레콤, 알파벳(구글) | 플랫폼·통신 혼재 |
| **유틸리티** | 한국전력, 넥스트에라 | 고배당·방어적 |
| **부동산** | 삼성리츠, 프롤로지스 | 금리에 역방향 |

#### ② KSIC (한국표준산업분류)

통계청이 관리하는 **국내 표준 산업 분류**. 사업자등록증·통계 신고의 공식 기준입니다.

| 대분류 | 설명 | 코드 |
|--------|------|------|
| 농업·임업·어업 | 1차 산업 | A |
| 광업 | 자원 채굴 | B |
| 제조업 | 가공·생산 | C |
| 전기·가스·증기 | 에너지 공급 | D |
| 건설업 | 토목·건축 | F |
| 도·소매업 | 유통 | G |
| 금융·보험업 | 금융 서비스 | K |
| 정보통신업 | IT·소프트웨어 | J |

> 💡 **사업자등록 시 업종·업태 기재**: 세무서에 사업자등록을 할 때 업종(KSIC 코드)과 업태(영업 형태 서술)를 모두 기재합니다. 이것이 세금 신고와 통계의 기초가 됩니다.

### C. 기업의 가치사슬 (Value Chain)

마이클 포터가 제시한 **가치사슬**은 기업이 가치를 창출하는 활동의 흐름입니다.

```
[제조업 가치사슬 예시 — 현대차]

원재료 조달 → 부품 제조 → 완성차 조립 → 유통·판매 → 애프터서비스
(POSCO 철강)  (현대모비스)  (현대차 공장)  (딜러십·현대캐피탈)  (블루핸즈)
     ↑                                                      ↑
  업스트림(Upstream)                                  다운스트림(Downstream)
```

| 포지션 | 의미 | 예시 |
|--------|------|------|
| **업스트림** | 원재료·부품 공급 단계 | 철강, 반도체 소재, 석유 채굴 |
| **미드스트림** | 가공·제조·운반 | 정유, 완성차 조립, 물류 |
| **다운스트림** | 소비자와 가장 가까운 단계 | 소매, 유통, 서비스 |

> 💡 **퀀트 활용**: 원자재 가격 상승 → 업스트림 기업 수혜, 다운스트림 기업 비용 압박. 가치사슬 위치를 알면 원자재·환율·금리 충격이 어느 기업에 얼마나 영향을 주는지 예측할 수 있습니다.

### D. B2B · B2C · B2G — 거래 상대방에 따른 분류

| 구분 | 의미 | 특징 | 대표 기업 |
|------|------|------|---------|
| **B2C** (Business to Consumer) | 기업 → 소비자 | 대량·소액 거래, 브랜드 중요 | 삼성전자(가전), 네이버, 스타벅스 |
| **B2B** (Business to Business) | 기업 → 기업 | 소량·고액 거래, 관계·신뢰 중요 | 삼성전자(반도체→애플), POSCO, SAP |
| **B2G** (Business to Government) | 기업 → 정부·공공기관 | 입찰·규제 중심, 장기 계약 | 한국항공우주(KAI), 두산중공업 |
| **C2C** (Consumer to Consumer) | 소비자 ↔ 소비자 | 플랫폼 중개, 신뢰 문제 | 당근마켓, 중고나라 |

> 💡 B2B 기업은 B2C보다 **매출 집중도(상위 고객 의존도)**가 높아 특정 고객사 이탈 리스크가 큽니다. 재무제표의 `매출처 집중도`를 확인해야 합니다.

### E. 경기 사이클과 업종 순환

거시경제 분석의 핵심은 **경기 국면**에 따라 어떤 업종이 유리한지 파악하는 것입니다.

```
[경기 사이클 4국면]

     회복기              확장기             둔화기              침체기
  (Recovery)          (Expansion)        (Slowdown)         (Recession)
      │                   │                  │                   │
```

### F. 매출·이익·현금흐름 — 숫자 읽는 순서

재무제표를 처음 볼 때 어디서부터 읽어야 할지 모르는 경우가 많습니다. 투자 분석용 핵심 숫자만 빠르게 확인하는 순서입니다.

```
Step 1. 매출 규모와 성장률     ← "이 회사 얼마짜리?"
         └── YoY 매출 성장률 = (올해 - 작년) / 작년 × 100

Step 2. 영업이익률              ← "본업은 얼마나 잘하나?"
         └── 영업이익률 = 영업이익 / 매출액 × 100

Step 3. 순이익률·EPS            ← "주주에게 돌아오는 몫"
         └── EPS = 당기순이익 / 발행주식수

Step 4. ROE (자기자본이익률)    ← "자본 효율성"
         └── ROE = 당기순이익 / 자기자본 × 100

Step 5. 부채비율                ← "재무 건전성"
         └── 부채비율 = 총부채 / 자기자본 × 100

Step 6. 영업현금흐름            ← "실제 현금이 들어오는가?"
         └── 이익과 현금흐름 괴리가 크면 분식 또는 운전자본 문제
```

```python
def quick_financial_check(
    revenue: float, op_income: float, net_income: float,
    equity: float, total_debt: float, shares: float,
    op_cashflow: float
) -> dict:
    """투자 분석용 핵심 재무비율 빠른 계산"""
    return {
        "영업이익률(%)":   round(op_income / revenue * 100, 1),
        "순이익률(%)":     round(net_income / revenue * 100, 1),
        "ROE(%)":          round(net_income / equity * 100, 1),
        "부채비율(%)":     round(total_debt / equity * 100, 1),
        "EPS(원)":         round(net_income / shares),
        "현금이익괴리":    round((op_cashflow - net_income) / net_income * 100, 1),
    }

# 예시: 가상 기업 A (단위: 억원, 주)
result = quick_financial_check(
    revenue=10_000, op_income=1_200, net_income=900,
    equity=6_000, total_debt=3_000, shares=1_000_000,
    op_cashflow=1_100
)

print("=== 가상 기업 A 핵심 재무비율 ===")
for k, v in result.items():
    print(f"  {k}: {v}")
```

### G. 시가총액과 기업가치 — 숫자의 의미

> 📖 **Wikipedia**: [시가총액](https://ko.wikipedia.org/wiki/시가총액) · [기업가치](https://ko.wikipedia.org/wiki/기업가치)

**시가총액(市價總額, Market Capitalization)**은 현재 주가 × 발행주식수로 계산하며, 주식 시장이 해당 기업에 부여하는 '가격표'입니다.

```
시가총액 = 주가 × 발행주식수 (총 주식 수)

예시: 삼성전자
  주가: 70,000원 × 발행주식수: 약 60억주 = 시가총액 약 420조원
```

| 지표 | 계산식 | 의미 |
|------|--------|------|
| **시가총액 (Market Cap)** | 주가 × 발행주식수 | 주식시장이 매긴 기업 가치 |
| **EV (기업가치)** | 시가총액 + 순차입금 | 회사 전체 인수 시 필요 금액 |
| **PER** | 주가 / EPS | 이익 대비 주가 수준 (낮을수록 저평가) |
| **PBR** | 주가 / BPS | 순자산 대비 주가 (1 미만 = 장부가보다 쌈) |
| **EV/EBITDA** | EV / EBITDA | 감가상각·이자·세금 전 이익 대비 기업가치 |

**시가총액 규모별 분류 (한국 기준)**

| 분류 | 시가총액 기준 | 특징 |
|------|------------|------|
| **대형주** | 10조원 이상 (KOSPI200 핵심) | 유동성 높음, 외국인 투자 집중 |
| **중형주** | 3조~10조원 | 성장성·안정성 균형 |
| **소형주** | 3천억~3조원 | 변동성 높음, 성장 잠재력 |
| **마이크로캡** | 3천억 미만 | 유동성 낮음, 분석 자료 희소 |

```python
# 시가총액 규모별 분류 (한국 기준)
def market_cap_tier(market_cap_billion: float) -> str:
    """시가총액(억원)으로 대·중·소형주 분류"""
    if market_cap_billion >= 100_000:   # 10조 이상
        return "대형주 (KOSPI200 핵심)"
    elif market_cap_billion >= 30_000:  # 3조 이상
        return "중형주"
    elif market_cap_billion >= 3_000:   # 3천억 이상
        return "소형주"
    else:
        return "마이크로캡"

examples = [
    ("삼성전자", 3_500_000),   # 350조
    ("현대차",    50_000),     # 5조
    ("코스닥 중소기업", 2_000), # 2천억
]
for name, cap in examples:
    print(f"{name:20}: {market_cap_tier(cap)} ({cap/10000:.0f}조원)")
```

**시가총액에 영향을 주는 주요 이벤트**

| 이벤트 | 주가 변화 | 주식수 변화 | 시가총액 변화 |
|--------|---------|-----------|------------|
| **주가 상승** | ↑ | 변동 없음 | ↑ |
| **주가 하락** | ↓ | 변동 없음 | ↓ |
| **유상증자** | 희석 (↓) | ↑ | 증가 (자금 유입) |
| **무상증자** | 비율만큼 ↓ | ↑ | 변동 없음 |
| **액면분할** | 비율만큼 ↓ | ↑ | **변동 없음** |
| **자사주 소각** | ↑ (주당 가치 상승) | ↓ | 감소 (자본 환원) |

> 💡 **액면분할과 시가총액**: 액면분할은 주당 가격을 낮추고 주식 수를 늘리지만, **시가총액(기업 전체 가치)은 변하지 않습니다**. 예를 들어 5:1 분할 시 주가는 1/5로 줄고 주식 수는 5배가 되어 시가총액은 동일합니다.

### 1. 매크로 분석의 정의와 투자에서의 중요성

> 📖 **Wikipedia**: [거시경제학](https://ko.wikipedia.org/wiki/거시경제학) · [하향식 투자](https://ko.wikipedia.org/wiki/하향식_투자)

**핵심 개념**

매크로(Macro) 분석은 **국가 경제 전체의 흐름**을 보는 하향식(Top-Down) 접근법입니다.

- 개별 기업을 분석하기 전에, 경제가 성장 중인지 수축 중인지부터 파악합니다.
- 금리·물가·환율·실업률 같은 거시경제 지표가 모든 자산 가격에 영향을 주기 때문입니다.

**투자에서 왜 중요한가**

거시경제 환경이 좋을 때는 아무 주식이나 사도 오를 수 있고, 환경이 나쁠 때는 좋은 기업 주식도 떨어질 수 있습니다. 즉 개별 기업 분석 전에 **시장 전체가 상승 국면인지 하락 국면인지**를 먼저 판단하는 것이 순서입니다.

```
Top-Down 분석 순서:
거시경제(매크로) → 산업 → 기업(개별 종목)
```

### 3. 금리와 주가·채권의 관계

> 📖 **Wikipedia**: [채권](https://ko.wikipedia.org/wiki/채권_(금융)) · [듀레이션](https://ko.wikipedia.org/wiki/듀레이션_(금융))

**금리와 주가의 관계**

| 금리 방향 | 주식시장 영향 | 이유 |
|-----------|---------------|------|
| 금리 ↑ | 주가 ↓ 압력 | 기업 이자 비용 증가, 미래 이익 할인율 상승, 채권 매력도 증가 |
| 금리 ↓ | 주가 ↑ 압력 | 저금리로 채권 매력 감소, 위험자산 선호 증가, 기업 차입 비용 감소 |

> 📺 [🎬 금리와 주식 관계](https://www.youtube.com/results?search_query=금리+주식+상관관계+투자+한국어)

**금리와 채권의 관계 (역방향)**

채권 가격과 금리는 반대로 움직입니다.

- 금리 ↑ → 기존 채권(낮은 쿠폰)의 매력 감소 → 채권 가격 ↓
- 금리 ↓ → 기존 채권의 상대적 매력 증가 → 채권 가격 ↑

> 📺 [🎬 채권 가격과 금리 역관계](https://www.youtube.com/results?search_query=채권가격+금리+역관계+한국어+설명)

**핵심 공식**

```python
# 채권 가격의 간단한 현재가치 계산
def bond_price(face_value, coupon_rate, market_rate, years):
    coupon = face_value * coupon_rate
    pv_coupons = sum(coupon / (1 + market_rate)**t for t in range(1, years+1))
    pv_principal = face_value / (1 + market_rate)**years
    return pv_coupons + pv_principal

# 시장금리가 오르면 채권 가격이 내려감을 확인
print(f"시장금리 3%일 때 가격: {bond_price(1000, 0.03, 0.03, 5):.2f}")
print(f"시장금리 5%일 때 가격: {bond_price(1000, 0.03, 0.05, 5):.2f}")
```

#### 5-2. 금리 데이터 제공 사이트/API 이용방법 정리

실습에서는 다음 순서로 데이터 수집기를 만듭니다.

```text
API 원천
  ├─ 한국은행 ECOS: 국내 정책금리/시장금리
  ├─ FRED: 미국 정책금리/국채금리/스프레드
  ├─ U.S. Treasury: 미국 재무부 금리 데이터
  ├─ KOFIA/KRX: 국내 채권·시장 보강 데이터
  └─ FinLife/Data.go.kr: 예금·대출 상품금리
        ↓

        ↓
date, country, source, series_id, series_name, tenor, value, unit
        ↓
CSV/Parquet 저장 + FastAPI 응답 + 시각화
```

**권장 표준 스키마**

| 컬럼 | 예시 | 설명 |
|---|---|---|

**한국은행 ECOS 예시**

ECOS는 URL 경로에 API 키, 응답 형식, 언어, 조회 범위, 통계표 코드, 주기, 기간, 항목 코드를 넣는 방식입니다. 실제 통계표/항목 코드는 ECOS 화면에서 검색해 확인합니다.

```python
import os
import requests
import pandas as pd

BOK_API_KEY = os.getenv("BOK_API_KEY")

def fetch_ecos_rate(stat_code, item_code, start="20200101", end="20241231", freq="D"):
    url = (
        "https://ecos.bok.or.kr/api/StatisticSearch/"
        f"{BOK_API_KEY}/json/kr/1/10000/{stat_code}/{freq}/{start}/{end}/{item_code}"
    )
    data = requests.get(url, timeout=20).json()
    rows = data.get("StatisticSearch", {}).get("row", [])
    df = pd.DataFrame(rows)
    if df.empty:
        return pd.DataFrame(columns=["date", "value"])
    return (
        df.assign(
            date=lambda x: pd.to_datetime(x["TIME"]),
            value=lambda x: pd.to_numeric(x["DATA_VALUE"], errors="coerce"),
        )[["date", "value"]]
        .dropna()
        .sort_values("date")
    )
```

**FRED 예시**

FRED는 `series_id` 중심으로 호출합니다. 금리 실습에서 자주 쓰는 코드는 `FEDFUNDS`, `DGS2`, `DGS10`, `DGS30`, `T10Y2Y`입니다.

```python
import os
import requests
import pandas as pd

FRED_API_KEY = os.getenv("FRED_API_KEY")

def fetch_fred_series(series_id, start="2020-01-01", end="2024-12-31"):
    url = "https://api.stlouisfed.org/fred/series/observations"
    params = {
        "series_id": series_id,
        "api_key": FRED_API_KEY,
        "file_type": "json",
        "observation_start": start,
        "observation_end": end,
    }
    data = requests.get(url, params=params, timeout=20).json()
    rows = data.get("observations", [])
    df = pd.DataFrame(rows)
    return (
        df.assign(
            date=lambda x: pd.to_datetime(x["date"]),
            value=lambda x: pd.to_numeric(x["value"].replace(".", pd.NA), errors="coerce"),
        )[["date", "value"]]
        .dropna()
        .sort_values("date")
    )
```

**U.S. Treasury Fiscal Data 예시**

미국 재무부 Fiscal Data API는 기본적으로 공개 REST API이며, `fields`, `filter`, `sort`, `format`, `page[size]` 같은 파라미터로 필요한 데이터만 가져옵니다.

```python
import requests
import pandas as pd

def fetch_treasury_avg_interest_rates(start="2020-01-01"):
    url = (
        "https://api.fiscaldata.treasury.gov/services/api/fiscal_service/"
        "v2/accounting/od/avg_interest_rates"
    )
    params = {
        "fields": "record_date,security_desc,avg_interest_rate_amt",
        "filter": f"record_date:gte:{start}",
        "sort": "record_date",
        "format": "json",
        "page[size]": "10000",
    }
    data = requests.get(url, params=params, timeout=20).json()
    df = pd.DataFrame(data.get("data", []))
    return df.assign(
        date=lambda x: pd.to_datetime(x["record_date"]),
        value=lambda x: pd.to_numeric(x["avg_interest_rate_amt"], errors="coerce"),
    )
```

```python
    end: str = Query("2024-12-31"),
):

    return {
        "source": source,
        "series_id": series_id,

            "max": float(df["value"].max()),
            "mean": float(df["value"].mean()),
```

처음에는 `PdfPages`로 차트형 PDF를 만들고, 이후 HTML 템플릿 기반 PDF로 확장하는 순서가 좋습니다.

```python
from matplotlib.backends.backend_pdf import PdfPages
import matplotlib.pyplot as plt

def build_rate_pdf_report(df, output_path="rate_report.pdf"):
    with PdfPages(output_path) as pdf:
        fig, ax = plt.subplots(figsize=(11, 6))
        ax.plot(df["date"], df["value"], color="crimson")
        ax.set_title("Interest Rate Trend")
        ax.set_ylabel("Rate (%)")
        ax.grid(True, alpha=0.3)
        pdf.savefig(fig, bbox_inches="tight")
        plt.close(fig)
    return output_path
```

#### 5-5. 데이터 품질 및 해석 유의사항

- 금리 데이터는 발표 주기와 휴일 정책이 다릅니다. 일별 데이터를 합칠 때는 `inner join`과 `outer join` 결과가 달라질 수 있습니다.
- `DGS10` 같은 미국 국채 금리는 시장금리이고, `FEDFUNDS`는 정책/은행간 금리 성격입니다. 서로 같은 금리처럼 비교하면 안 됩니다.
- 한국 국고채 금리와 미국 국채 금리는 통화, 물가, 성장률, 중앙은행 정책이 다르므로 단순 레벨 비교보다 방향성과 스프레드를 함께 봅니다.
- 결측치는 무조건 전일 값으로 채우지 말고, 원천 데이터가 휴장인지 미발표인지 먼저 구분합니다.
- 보고서에는 반드시 데이터 출처, 조회일, 시리즈 코드, 단위, 결측 처리 방법을 표시합니다.

### A. 사람과 법인 — 법적 주체의 구분

#### ① 자연인(個人) vs 법인(法人)

| 구분 | 자연인 (Natural Person) | 법인 (Legal Person / Corporation) |
|------|------------------------|-----------------------------------|
| 의미 | 태어나면서 권리·의무의 주체가 되는 사람 | 법이 사람처럼 권리·의무 능력을 부여한 **인위적 존재** |
| 사례 | 홍길동 씨 (개인) | 삼성전자 주식회사, 사단법인 XX협회 |
| 설립 방법 | 출생 | 설립 등기 (법원·등기소) |
| 사망/소멸 | 자연사 | 해산·청산 |
| 세금 | 소득세 (종합소득세·근로소득세) | 법인세 |
| 재산 | 본인 재산 = 개인 재산 | 법인 재산 ≠ 주주 재산 (분리) |
| 채무 책임 | 본인 전 재산으로 무한 책임 | 원칙적으로 법인 재산 한도 내 |

> 💡 법인이 **별도의 법적 인격**을 가진다는 것이 핵심입니다. 삼성전자가 빚을 져도, 이재용 회장이 개인 재산으로 갚을 의무는 없습니다(주주 유한 책임).

---

#### ② 개인사업자 vs 법인사업자

사업을 할 때는 **개인사업자**와 **법인사업자(회사)** 중 하나를 선택합니다.

| 구분 | 개인사업자 | 법인사업자 |
|------|-----------|-----------|
| 법적 주체 | 자연인(사장 본인) | 별도 법인 |
| 설립 절차 | 사업자 등록 (세무서, 간단) | 법인 설립 등기 (법원) + 사업자 등록 |
| 최저 자본금 | 없음 | 없음 (단, 주식회사는 실무상 최소 100만원 이상) |
| 소득세 vs 법인세 | **종합소득세** (6~45% 누진) | **법인세** (9~24%, 낮은 세율) |
| 대표자 급여 | 비용 처리 **불가** (사장 소득 = 사업 소득) | 비용 처리 **가능** (급여로 처리) |
| 채무 책임 | **무한 책임** — 사업 빚 = 개인 빚 | **유한 책임** — 출자액 한도 |
| 대출·투자 유치 | 상대적으로 어려움 | 주식 발행·외부 투자 유치 용이 |
| 폐업 절차 | 간단 (폐업 신고) | 복잡 (청산·등기 말소) |
| 적합 상황 | 소규모 자영업·초기 1인 사업 | 성장을 목표로 한 사업, 투자 유치 필요 시 |

> 💡 **절세 전략**: 연 소득이 약 5,000만원을 초과하면 소득세 세율이 법인세보다 높아지기 시작합니다. 이 시점에 법인 전환을 검토하는 경우가 많습니다.

### B. 회사의 종류 — 책임 구조에 따른 분류

상법은 회사를 **책임 방식**에 따라 5종으로 구분합니다.

| 회사 형태 | 사원의 책임 | 특징 | 실제 사용 |
|-----------|-----------|------|----------|
| **합명회사** | **전원 무한 책임** | 소수 신뢰 관계, 인적 회사 | 거의 없음 |
| **합자회사** | 무한책임사원 + 유한책임사원 혼합 | 투자자 보호 일부 가능 | 드묾 |
| **유한책임회사** | **전원 유한 책임** | 미국 LLC와 유사, 내부 자유도 높음 | 스타트업·VC |
| **유한회사** | **전원 유한 책임** | 사원 50인 이하, 비공개 | 중소기업 |
| **주식회사** | **전원 유한 책임** | 주식 발행, 공개·상장 가능 | **가장 보편적** |

---

#### ① 무한책임 (Unlimited Liability)

사업체의 채무를 **개인 재산으로도 끝까지 갚아야** 하는 책임 구조입니다.

```
사업 빚 1억원 발생
  └── 무한책임 → 개인 통장·집·자동차까지 전부 상환 의무
```

- 해당: 개인사업자, 합명회사 사원, 합자회사 무한책임사원
- 위험: 사업 실패 시 개인 파산으로 이어질 수 있음

#### ② 유한책임 (Limited Liability)

**출자한 금액 한도 내에서만** 책임지는 구조입니다.

```
주식 1,000만원어치 투자
  └── 유한책임 → 최대 손실 = 1,000만원 (개인 재산 추가 손실 없음)
```

- 해당: 주식회사 주주, 유한회사·유한책임회사 사원
- 의의: 위험 부담을 투자액으로 제한 → 대규모 자본 조달 가능

> 💡 유한책임 덕분에 수백만 명이 삼성전자 주식을 살 수 있습니다. 회사가 망해도 주가가 0원이 될 뿐, 개인이 회사 빚을 갚지 않아도 됩니다.

### C. 주식회사 (株式會社, Corporation)

#### ① 주식회사의 구조

```
주주 (출자자·소유자)
  │  출자 (주식 매입)
  ▼
주식회사 (법인)
  │  선임
  ▼
이사회 (Board of Directors)
  │  경영 위임
  ▼
대표이사 (CEO) + 경영진
```

| 기관 | 역할 |
|------|------|
| **주주총회** | 최고 의사결정 기관. 이사 선임, 합병, 정관 변경 등 |
| **이사회** | 경영 감독. 주요 업무 결의 |
| **대표이사** | 회사 대표, 일상 경영 집행 |
| **감사** | 이사 업무·회계 감시 |

#### ② 주식 (株式, Share)

| 항목 | 내용 |
|------|------|
| 의미 | 주식회사 소유권(지분)을 균등하게 나눈 단위 |
| 종류 | 보통주 (의결권 O) / 우선주 (배당 우선, 의결권 X) |
| 액면가 | 정관에 기재된 주당 명목 금액 (보통 100원·500원·5,000원) |
| 시가 | 시장에서 거래되는 실제 가격 |
| 배당 | 이익 일부를 주주에게 분배 |
| 의결권 | 1주 = 1표 (보통주 기준) |

#### ③ 상장 vs 비상장

| 구분 | 상장(公開)회사 | 비상장(私的)회사 |
|------|--------------|----------------|
| 주식 거래 | 거래소(KRX)에서 자유롭게 매매 | 장외·개인간 거래 |
| 공시 의무 | DART 공시 의무 (강함) | 거의 없음 |
| 자금 조달 | IPO·유상증자로 불특정 다수에게 | 제한적 |
| 대표 예시 | 삼성전자, 카카오 | 카카오 (상장 전), 비상장 중소기업 |

### D. 세금 — 개인과 법인의 차이

#### ① 소득세 (所得稅, Income Tax) — 개인

개인의 소득에 부과하는 세금. **누진세율** 구조입니다.

| 과세표준 | 세율 | 누진공제 |
|---------|------|---------|
| 1,400만원 이하 | **6%** | — |
| 1,400~5,000만원 | **15%** | 126만원 |
| 5,000~8,800만원 | **24%** | 576만원 |
| 8,800만원~1.5억원 | **35%** | 1,544만원 |
| 1.5억~3억원 | **38%** | 1,994만원 |
| 3억~5억원 | **40%** | 2,594만원 |
| 5억~10억원 | **42%** | 3,594만원 |
| 10억원 초과 | **45%** | 6,594만원 |

> 지방소득세 10% 추가 → 최고 세율 실질 **49.5%**

```python
def income_tax(taxable_income: int) -> int:
    """종합소득세 계산 (2024년 기준, 지방소득세 제외)"""
    brackets = [
        (14_000_000,  0.06,         0),
        (50_000_000,  0.15, 1_260_000),
        (88_000_000,  0.24, 5_760_000),
        (150_000_000, 0.35,15_440_000),
        (300_000_000, 0.38,19_940_000),
        (500_000_000, 0.40,25_940_000),
        (1_000_000_000,0.42,35_940_000),
        (float('inf'), 0.45,65_940_000),
    ]
    for limit, rate, deduction in brackets:
        if taxable_income <= limit:
            return int(taxable_income * rate - deduction)
    return 0

for income in [30_000_000, 80_000_000, 200_000_000, 500_000_000]:
    tax = income_tax(income)
    rate = tax / income * 100
    print(f"소득 {income:>12,}원 → 세금 {tax:>11,}원 (실효세율 {rate:.1f}%)")
```

**실행 결과:**
```
소득   30,000,000원 → 세금   3,240,000원 (실효세율 10.8%)
소득   80,000,000원 → 세금  13,440,000원 (실효세율 16.8%)
소득  200,000,000원 → 세금  56,060,000원 (실효세율 28.0%)
소득  500,000,000원 → 세금 174,060,000원 (실효세율 34.8%)
```

#### ② 법인세 (法人稅, Corporate Tax) — 법인

법인의 각 사업연도 **소득**에 부과하는 세금.

| 과세표준 | 세율 |
|---------|------|
| 2억원 이하 | **9%** |
| 2억~200억원 | **19%** |
| 200억~3,000억원 | **21%** |
| 3,000억원 초과 | **24%** |

> 지방소득세 10% 추가 → 최고 실질 세율 **26.4%**

```python
def corporate_tax(taxable_income: int) -> int:
    """법인세 계산 (2024년 기준, 지방소득세 제외)"""
    brackets = [
        (200_000_000,       0.09,           0),
        (20_000_000_000,    0.19,  20_000_000),
        (300_000_000_000,   0.21,  60_000_000),
        (float('inf'),      0.24, 150_000_000),
    ]
    for limit, rate, deduction in brackets:
        if taxable_income <= limit:
            return int(taxable_income * rate - deduction)
    return 0

for income in [100_000_000, 500_000_000, 5_000_000_000, 400_000_000_000]:
    tax = corporate_tax(income)
    rate = tax / income * 100
    label = f"{income/100_000_000:.0f}억원"
    print(f"과세표준 {label:>8} → 법인세 {tax:>14,}원 (실효세율 {rate:.1f}%)")
```

**실행 결과:**
```
과세표준      1억원 → 법인세      9,000,000원 (실효세율  9.0%)
과세표준      5억원 → 법인세     75,000,000원 (실효세율 15.0%)
과세표준     50억원 → 법인세    930,000,000원 (실효세율 18.6%)
과세표준   4000억원 → 법인세 85,050,000,000원 (실효세율 21.3%)
```

#### ③ 소득세 vs 법인세 비교

```python
import matplotlib.pyplot as plt

incomes = np.linspace(1e7, 5e8, 500)

plt.figure(figsize=(10, 5))
plt.plot(incomes / 1e8, personal,  label='소득세 (개인)', color='#e74c3c')

plt.xlabel('과세표준 (억원)'); plt.ylabel('실효세율 (%)')

```

### E. 세무 기초 — 세금의 종류와 구조

#### ① 세금 분류 체계

```
세금
├── 국세 (국가)
│   ├── 직접세 (납세자 = 담세자)
│   │   ├── 소득세 (개인 소득)
│   │   ├── 법인세 (법인 소득)
│   │   ├── 상속세 (사망으로 인한 재산 이전)
│   │   └── 증여세 (생전 재산 증여)
│   └── 간접세 (납세자 ≠ 담세자)
│       ├── 부가가치세 (VAT, 소비세)
│       └── 개별소비세 (사치품·유류)
└── 지방세 (지자체)
    ├── 취득세 (부동산·차량 취득)
    ├── 재산세 (토지·건물 보유)
    └── 지방소득세 (소득세·법인세의 10%)
```

#### ② 부가가치세 (VAT, Value Added Tax)

소비자가 물건을 살 때 내는 **소비세**. 사업자는 **대리 징수·납부**합니다.

| 항목 | 내용 |
|------|------|
| 세율 | **10%** (면세·영세율 제외) |
| 납세자 | 사업자 (실질 부담은 최종 소비자) |
| 과세 기간 | 6개월 (연 2회 신고·납부) |
| 매출세액 | 매출액 × 10% → 고객으로부터 거두는 세금 |
| 매입세액 | 매입액 × 10% → 구매 시 낸 세금 (공제 가능) |
| 납부액 | **매출세액 − 매입세액** |

```python
# 부가가치세 계산 예시
sales_revenue  = 10_000_000  # 매출 1,000만원 (VAT 포함 1,100만원)
purchase_cost  =  6_000_000  # 매입 600만원 (VAT 포함 660만원)

output_vat = sales_revenue  * 0.10   # 매출세액 100만원
input_vat  = purchase_cost * 0.10   # 매입세액 60만원
vat_payable = output_vat - input_vat # 납부 부가세 40만원

print(f"매출세액(매출 VAT):  {output_vat:>10,.0f}원")
print(f"매입세액(매입 VAT):  {input_vat:>10,.0f}원")
print(f"납부 부가가치세:     {vat_payable:>10,.0f}원")
```

#### ③ 원천징수 (源泉徵收, Withholding Tax)

소득을 **지급하는 자**가 세금을 미리 떼어 납부하는 제도.

| 소득 종류 | 원천징수 세율 | 적용 예시 |
|-----------|-------------|---------|
| 근로소득 | 간이세액표 적용 | 월급에서 소득세 선공제 |
| 이자소득 | **15.4%** (소득세 14% + 지방 1.4%) | 은행 예금 이자 |
| 배당소득 | **15.4%** | 주식 배당금 |
| 사업소득(프리랜서) | **3.3%** | 강사료·원고료 |
| 기타소득 | **22%** (필요경비 60% 인정 후) | 강연료·복권 당첨금 |

> 💡 **금융소득 종합과세**: 이자+배당 합계가 연 2,000만원 초과 시 다른 소득과 합산해 종합소득세 신고·납부 의무 발생.

### F. 회계 기초 — 재무제표 이해

#### ① 회계의 목적

회계(Accounting)는 **기업의 경제 활동을 측정·기록·보고**하는 것입니다.

| 구분 | 재무회계 | 세무회계 | 관리회계 |
|------|---------|---------|---------|
| 목적 | 외부 이해관계자 보고 | 세금 계산 | 내부 의사결정 지원 |
| 기준 | K-IFRS / K-GAAP | 세법 | 회사 자체 기준 |
| 보고서 | 재무제표 | 세무조정계산서 | 예산·원가 보고서 |
| 강제성 | 법 의무 (상장사) | 법 의무 | 자율 |

#### ② 재무제표 3대 보고서

| 보고서 | 핵심 질문 | 주요 항목 |
|--------|----------|----------|
| **재무상태표** (B/S) | "지금 재산이 얼마고 빚이 얼마냐?" | 자산 = 부채 + 자본 |
| **손익계산서** (I/S) | "이번에 얼마 벌었냐?" | 매출 → 영업이익 → 당기순이익 |
| **현금흐름표** (C/F) | "실제 현금이 얼마나 들어왔냐?" | 영업·투자·재무 활동 현금 |

```
[재무상태표 구조]
자산 (Assets)                부채 (Liabilities)
  ├── 유동자산                  ├── 유동부채 (1년 이내)
  │   ├── 현금·예금              └── 비유동부채 (1년 초과)
  │   ├── 매출채권             자본 (Equity)
  │   └── 재고자산               ├── 자본금 (주주 출자)
  └── 비유동자산                 ├── 이익잉여금 (누적 순이익)
      ├── 유형자산               └── 기타포괄손익누계액
      └── 무형자산
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
         자산 합계  =  부채 합계 + 자본 합계  (항등식)
```

#### ③ 손익계산서 구조

```
매출액 (Revenue)
  − 매출원가 (Cost of Goods Sold)
━━━━━━━━━━━━━━━━━━
= 매출총이익 (Gross Profit)
  − 판매비와관리비 (SG&A)
━━━━━━━━━━━━━━━━━━
= 영업이익 (Operating Income / EBIT)
  + 영업외수익 (이자수익·배당 등)
  − 영업외비용 (이자비용·환차손 등)
━━━━━━━━━━━━━━━━━━
= 세전이익 (EBT)
  − 법인세 비용
━━━━━━━━━━━━━━━━━━
= 당기순이익 (Net Income)
```

#### ④ 회계 기본 등식과 분개 원리

```
자산 = 부채 + 자본  (재무상태표 항등식)
수익 − 비용 = 이익  (손익계산서)
```

**분개(Journal Entry) 예시:**

| 거래 | 차변 (Debit) | 대변 (Credit) |
|------|------------|--------------|
| 현금 100만원 매출 | 현금 100만원 ↑ | 매출 100만원 ↑ |
| 임직원 급여 50만원 지급 | 급여비용 50만원 ↑ | 현금 50만원 ↓ |
| 설비 200만원 구입 (현금) | 유형자산 200만원 ↑ | 현금 200만원 ↓ |
| 은행 대출 1,000만원 | 현금 1,000만원 ↑ | 차입금 1,000만원 ↑ |

> 💡 **차변 = 자산·비용 증가 / 대변 = 부채·자본·수익 증가**. 양쪽 합계는 항상 같아야 합니다(복식부기).

### G. 세무조정 — 회계이익 vs 과세소득

재무회계(K-IFRS) 기준 **회계이익**과 세법 기준 **과세소득**은 다릅니다.

```
회계이익 (당기순이익)
  + 익금산입 (세법은 수익 인정, 회계는 미인식)
  − 손금산입 (세법은 비용 인정, 회계는 미인식)
  − 익금불산입 (회계는 수익, 세법은 비수익)
  + 손금불산입 (회계는 비용, 세법은 불인정)
━━━━━━━━━━━━━━━━━━━━━━━━━━
= 각 사업연도 소득 (과세표준)
  × 법인세율
= 산출 법인세액
  − 세액공제·감면
= 최종 납부 법인세
```

| 항목 | 예시 | 방향 |
|------|------|------|
| **손금불산입** | 업무와 무관한 접대비 초과분, 임원 상여금 한도 초과 | 과세소득 **증가** |
| **익금불산입** | 자회사 배당금 익금불산입 (이중과세 방지) | 과세소득 **감소** |
| **감가상각비** | 회계 내용연수 vs 세법 내용연수 차이 | 임시 차이 발생 |
| **대손충당금** | 회계는 추정·설정, 세법은 실제 대손 확정 시 인정 | 임시 차이 발생 |

> 💡 **이연법인세(Deferred Tax)**: 회계이익과 과세소득의 차이(임시 차이)로 인해 미래에 낼 세금이나 돌려받을 세금을 재무상태표에 미리 계상하는 항목입니다.

### H. 배당과 이중과세

법인이 이익을 내면 **법인세 → 배당 → 배당소득세** 순으로 세금이 두 번 부과됩니다.

```
법인 이익 1,000만원
  − 법인세 (예: 9%) 90만원
  = 세후이익 910만원 → 주주에게 배당
    − 배당소득세 15.4%: 140만원
    = 주주 실수령 770만원

이중과세 합산 부담: 90 + 140 = 230만원 (실효 23%)
```

> 💡 이 문제를 완화하기 위해 **배당세액공제** 제도가 있습니다. 종합소득세 신고 시 배당금에 대해 11%를 세액공제 해줍니다.

### J. 퀀트·투자 분석에서 세금이 중요한 이유

| 분석 항목 | 세금 연관성 |
|-----------|-----------|
| **EPS (주당순이익)** | 법인세 차감 후 당기순이익 기준 → 실효 세율 변화가 EPS에 직접 영향 |
| **FCF (잉여현금흐름)** | 세후 영업이익(NOPAT) 기반 → 법인세율 변화 시 DCF 밸류에이션 전체 변화 |
| **배당수익률** | 배당소득세 15.4% 차감 후 실질 수익 계산 필요 |
| **PER 비교** | 국가별 법인세율 차이 → 세후이익(분모) 달라짐 → 단순 PER 비교 왜곡 가능 |
| **이연법인세** | 재무상태표 항목. 갑자기 커지면 과세소득과 회계이익 간 큰 차이 발생 신호 |
| **세금 혜택** | R&D 세액공제, 중소기업 특별 감면 → 실효 법인세율이 명목보다 훨씬 낮을 수 있음 |

```python
# 실효 법인세율 계산 (DART 데이터 활용)
def effective_tax_rate(pretax_income: int, income_tax_expense: int) -> float:
    """실효 법인세율 = 법인세비용 / 세전이익"""
    if pretax_income <= 0:
        return float('nan')
    return income_tax_expense / pretax_income

# 예: 삼성전자 2023년 (가상 수치)
pretax  = 6_000_000_000_000   # 세전이익 6조
tax_exp = 1_200_000_000_000   # 법인세비용 1.2조
etr = effective_tax_rate(pretax, tax_exp)
nominal = 0.24  # 명목 법인세율 (3,000억 초과 구간)

print(f"명목 법인세율:  {nominal*100:.1f}%")
print(f"실효 법인세율:  {etr*100:.1f}%")
print(f"차이: {(nominal - etr)*100:.1f}%p → R&D 공제·세액감면 효과")
```
