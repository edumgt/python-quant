### 1. 매크로 분석의 정의와 투자에서의 중요성

**핵심 개념**

매크로(Macro) 분석은 **국가 경제 전체의 흐름**을 보는 하향식(Top-Down) 접근법입니다.

- 개별 기업을 분석하기 전에, 경제가 성장 중인지 수축 중인지부터 파악합니다.
- 금리·물가·환율·실업률 같은 거시경제 지표가 모든 자산 가격에 영향을 주기 때문입니다.

**투자에서 왜 중요한가**

거시경제 환경이 좋을 때는 아무 주식이나 사도 오를 수 있고, 환경이 나쁠 때는 좋은 기업 주식도 떨어질 수 있습니다. 즉 개별 기업 분석 전에 **시장 전체가 상승 국면인지 하락 국면인지**를 먼저 판단하는 것이 순서입니다.

```
Top-Down 분석 순서:
거시경제(매크로) → 산업 → 기업(개별 종목)
```

> 📺 [🎬 탑다운 분석 투자전략](https://www.youtube.com/results?search_query=탑다운+분석+투자전략+한국어)

**핵심 거시경제 지표 요약**

| 지표 | 의미 | 주요 발표 기관 |
|------|------|----------------|
| GDP 성장률 | 경제 규모 성장 속도 | 한국은행, Bureau of Economic Analysis(미국) |
| 금리 | 돈의 가격 | 한국은행, 연준(Fed) |
| CPI(소비자물가) | 인플레이션 수준 | 통계청, BLS(미국) |
| 실업률 | 노동시장 상황 | 통계청, BLS(미국) |
| 환율 | 통화 간 교환 비율 | 외환시장 |

### 2. 금리(기준금리, 시장금리) 이해

**기준금리(Base Rate)**

중앙은행이 결정하는 정책 금리로, 모든 금리의 기준점이 됩니다.

- 한국: 한국은행 금융통화위원회가 연 8회 결정
- 미국: 연준(Fed) FOMC가 연 8회 결정

> 📺 [🎬 기준금리란 무엇인가](https://www.youtube.com/results?search_query=기준금리+뜻+한국은행+설명+한국어)

**시장금리(Market Rate)**

시장 참가자들의 수요·공급으로 결정되는 금리입니다.

- **단기금리**: CD금리, 콜금리 (수일~수개월)
- **장기금리**: 국채 10년물 금리 (10년)
- **스프레드**: 장단기 금리 차이 → 경기 예측에 사용

> 📺 [🎬 시장금리 장단기금리 스프레드](https://www.youtube.com/results?search_query=시장금리+장단기+스프레드+경기예측+한국어)

**금리 구조 한눈에 보기**

```
기준금리 ↑
    │
    ├─ CD금리(91일)  ← 단기 시장금리
    ├─ 국고채 3년물  ← 중기 기준
    └─ 국고채 10년물 ← 장기 기준, 경기 전망 반영
```

### 3. 금리와 주가·채권의 관계

**금리와 주가의 관계**

| 금리 방향 | 주식시장 영향 | 이유 |
|-----------|---------------|------|
| 금리 ↑ | 주가 ↓ 압력 | 기업 이자 비용 증가, 미래 이익 할인율 상승, 채권 매력도 증가 |
| 금리 ↓ | 주가 ↑ 압력 | 저금리로 채권 매력 감소, 위험자산 선호 증가, 기업 차입 비용 감소 |

> 📺 [🎬 금리와 주식 관계](https://www.youtube.com/results?search_query=금리+주식+상관관계+투자+한국어)

**금리와 채권의 관계 (역방향)**

채권 가격과 금리는 반대로 움직입니다.

- 금리 ↑ → 기존 채권(낮은 쿠폰)의 매력 감소 → 채권 가격 ↓
- 금리 ↓ → 기존 채권의 상대적 매력 증가 → 채권 가격 ↑

> 📺 [🎬 채권 가격과 금리 역관계](https://www.youtube.com/results?search_query=채권가격+금리+역관계+한국어+설명)

**핵심 공식**

```python
# 채권 가격의 간단한 현재가치 계산
def bond_price(face_value, coupon_rate, market_rate, years):
    coupon = face_value * coupon_rate
    pv_coupons = sum(coupon / (1 + market_rate)**t for t in range(1, years+1))
    pv_principal = face_value / (1 + market_rate)**years
    return pv_coupons + pv_principal

# 시장금리가 오르면 채권 가격이 내려감을 확인
print(f"시장금리 3%일 때 가격: {bond_price(1000, 0.03, 0.03, 5):.2f}")
print(f"시장금리 5%일 때 가격: {bond_price(1000, 0.03, 0.05, 5):.2f}")
```

### 4. 한국은행 및 연준(Fed) 금리 결정 메커니즘

**한국은행 금융통화위원회(금통위)**

- 총재 포함 7명으로 구성
- 연 8회 정기 회의 (매년 1, 2, 4, 5, 7, 8, 10, 11월)
- 결정 근거: CPI, GDP 성장률, 고용 상황, 부동산 시장

> 📺 [🎬 한국은행 금통위 금리결정 과정](https://www.youtube.com/results?search_query=한국은행+금통위+기준금리+결정+한국어)

**미국 연방공개시장위원회(FOMC)**

- Fed 의장 포함 12명 투표권자 (지역 연준 총재 4명은 순환)
- 연 8회 정기 회의
- 결정 근거: 물가 안정(CPI 2% 목표) + 완전고용 이중 목표(Dual Mandate)

> 📺 [🎬 미국 연준 FOMC 금리결정 파월](https://www.youtube.com/results?search_query=FOMC+연준+금리결정+파월+한국어+해설)

**금리 결정 신호 읽기**

| 발언·신호 | 의미 |
|-----------|------|
| 매파(Hawkish) | 금리 인상 선호, 인플레이션 억제 우선 |
| 비둘기파(Dovish) | 금리 인하 선호, 경기 부양 우선 |
| 점도표(Dot Plot) | FOMC 위원별 금리 전망 분포도 |

4. KRX: Data Marketplace 회원가입 → 인증키 신청 → API 서비스 목록 검색 → 원하는 서비스 활용신청 → 승인 후 사용합니다.
5. 공공데이터/금융상품 API: 제공기관의 활용신청 화면에서 서비스 키를 발급받고, 호출 제한과 상업적 이용 조건을 확인합니다.

**유사 정보 조회 방법**

같은 금리라도 출처와 정의가 다르면 값이 다를 수 있습니다. 아래처럼 반드시 데이터 정의를 같이 기록합니다.

| 비교 항목 | ECOS | FRED 또는 Treasury | 확인 질문 |
|---|---|---|---|
| 정책금리 | 한국은행 기준금리 | `FEDFUNDS`, `DFEDTARU`, `DFEDTARL` | 일별/월별 중 무엇인가? 목표범위인가, 실효금리인가? |
| 장기금리 | 국고채 10년 | `DGS10` | 종가/고시/평균 중 무엇인가? 휴일 처리는 어떻게 하는가? |
| 단기금리 | CD 91일, 콜금리 | `DGS3MO`, `SOFR` | 은행간 금리인가, 국채금리인가? |
| 스프레드 | 국고채 10년 - 3년 | `T10Y2Y`, `DGS10 - DGS2` | 역전 구간을 경기 신호로 해석할 수 있는가? |

**한국은행 ECOS 예시**

ECOS는 URL 경로에 API 키, 응답 형식, 언어, 조회 범위, 통계표 코드, 주기, 기간, 항목 코드를 넣는 방식입니다. 실제 통계표/항목 코드는 ECOS 화면에서 검색해 확인합니다.

```python
import os
import requests
import pandas as pd

BOK_API_KEY = os.getenv("BOK_API_KEY")

def fetch_ecos_rate(stat_code, item_code, start="20200101", end="20241231", freq="D"):
    url = (
        "https://ecos.bok.or.kr/api/StatisticSearch/"
        f"{BOK_API_KEY}/json/kr/1/10000/{stat_code}/{freq}/{start}/{end}/{item_code}"
    )
    data = requests.get(url, timeout=20).json()
    rows = data.get("StatisticSearch", {}).get("row", [])
    df = pd.DataFrame(rows)
    if df.empty:
        return pd.DataFrame(columns=["date", "value"])
    return (
        df.assign(
            date=lambda x: pd.to_datetime(x["TIME"]),
            value=lambda x: pd.to_numeric(x["DATA_VALUE"], errors="coerce"),
        )[["date", "value"]]
        .dropna()
        .sort_values("date")
    )
```

**FRED 예시**

FRED는 `series_id` 중심으로 호출합니다. 금리 실습에서 자주 쓰는 코드는 `FEDFUNDS`, `DGS2`, `DGS10`, `DGS30`, `T10Y2Y`입니다.

```python
import os
import requests
import pandas as pd

FRED_API_KEY = os.getenv("FRED_API_KEY")

def fetch_fred_series(series_id, start="2020-01-01", end="2024-12-31"):
    url = "https://api.stlouisfed.org/fred/series/observations"
    params = {
        "series_id": series_id,
        "api_key": FRED_API_KEY,
        "file_type": "json",
        "observation_start": start,
        "observation_end": end,
    }
    data = requests.get(url, params=params, timeout=20).json()
    rows = data.get("observations", [])
    df = pd.DataFrame(rows)
    return (
        df.assign(
            date=lambda x: pd.to_datetime(x["date"]),
            value=lambda x: pd.to_numeric(x["value"].replace(".", pd.NA), errors="coerce"),
        )[["date", "value"]]
        .dropna()
        .sort_values("date")
    )
```

**U.S. Treasury Fiscal Data 예시**

미국 재무부 Fiscal Data API는 기본적으로 공개 REST API이며, `fields`, `filter`, `sort`, `format`, `page[size]` 같은 파라미터로 필요한 데이터만 가져옵니다.

```python
import requests
import pandas as pd

def fetch_treasury_avg_interest_rates(start="2020-01-01"):
    url = (
        "https://api.fiscaldata.treasury.gov/services/api/fiscal_service/"
        "v2/accounting/od/avg_interest_rates"
    )
    params = {
        "fields": "record_date,security_desc,avg_interest_rate_amt",
        "filter": f"record_date:gte:{start}",
        "sort": "record_date",
        "format": "json",
        "page[size]": "10000",
    }
    data = requests.get(url, params=params, timeout=20).json()
    df = pd.DataFrame(data.get("data", []))
    return df.assign(
        date=lambda x: pd.to_datetime(x["record_date"]),
        value=lambda x: pd.to_numeric(x["avg_interest_rate_amt"], errors="coerce"),
    )
```

def build_rate_pdf_report(df, output_path="rate_report.pdf"):

#### 5-5. 데이터 품질 및 해석 유의사항

- 금리 데이터는 발표 주기와 휴일 정책이 다릅니다. 일별 데이터를 합칠 때는 `inner join`과 `outer join` 결과가 달라질 수 있습니다.
- `DGS10` 같은 미국 국채 금리는 시장금리이고, `FEDFUNDS`는 정책/은행간 금리 성격입니다. 서로 같은 금리처럼 비교하면 안 됩니다.
- 한국 국고채 금리와 미국 국채 금리는 통화, 물가, 성장률, 중앙은행 정책이 다르므로 단순 레벨 비교보다 방향성과 스프레드를 함께 봅니다.
- 결측치는 무조건 전일 값으로 채우지 말고, 원천 데이터가 휴장인지 미발표인지 먼저 구분합니다.
- 보고서에는 반드시 데이터 출처, 조회일, 시리즈 코드, 단위, 결측 처리 방법을 표시합니다.
