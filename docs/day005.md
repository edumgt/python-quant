### 6. 예외처리 (try, except, finally)

**쉽게 이해하기**
- **예외처리 (try, except, finally)**는 `조건 점검 → 예외 대응 → 손실 제한` 순서로 설계하면 안정성이 높아집니다.
- 정상 시나리오뿐 아니라 실패 시나리오(네트워크 오류, 과도한 포지션)도 함께 테스트하세요.

**미니 예시**
```python
# 예외처리 (try, except, finally) 미니 예시
max_position = 100000
order_amount = 120000
try:
    if order_amount > max_position:
        raise ValueError('리스크 한도 초과')
    print('주문 실행')
except ValueError as e:
    print('주문 차단:', e)
```

### 7. 실습: 포트폴리오 클래스 계층 구조 설계

**쉽게 이해하기**
- **실습: 포트폴리오 클래스 계층 구조 설계** 개념은 `자산 비중 정하기 → 성과 계산 → 비중 조정` 흐름으로 보면 쉽습니다.
- 한 번에 복잡하게 최적화하지 말고 2개 자산 예제로 먼저 감을 잡으세요.

**미니 예시**
```python
# 실습: 포트폴리오 클래스 계층 구조 설계 미니 예시
weights = {'stock': 0.6, 'bond': 0.4}
asset_return = {'stock': 0.12, 'bond': 0.04}
portfolio_return = sum(weights[a] * asset_return[a] for a in weights)
print(f'기대수익률: {portfolio_return:.2%}')
```
